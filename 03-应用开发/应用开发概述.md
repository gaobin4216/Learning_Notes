# Web 前后端与桌面应用开发

从 Web 全栈到原生桌面应用的入门知识体系。

---

## 两大方向对比

| | Web 开发 | 桌面开发 |
|---|---|---|
| **运行环境** | 浏览器 | 操作系统（Windows/Mac/Linux） |
| **核心技术** | HTML + CSS + JavaScript | C++ (Qt) 或 C# (WPF/WinUI3) |
| **跨平台** | 天然跨平台 | Qt 跨平台，WPF/WinUI3 仅 Windows |
| **部署方式** | 不用安装，刷新即更新 | 需要安装，更新需重新分发 |
| **性能** | 受浏览器沙箱限制 | 直接调用系统资源 |
| **适合场景** | 网站、Web 应用、小程序 | 工具软件、工业软件、企业应用 |

---

## 桌面技术选型

| | Qt | WPF | WinUI3 |
|---|---|---|---|
| **语言** | C++ | C# | C# |
| **跨平台** | Win/Mac/Linux/嵌入式 | 仅 Windows | 仅 Windows |
| **UI 风格** | 传统 + 现代 (QML) | 经典 Windows | Fluent Design (Win11) |
| **学习难度** | 高（C++ 门槛） | 中（成熟，资料多） | 中（较新，坑略多） |
| **成熟度** | 非常成熟 | 非常成熟 (18 年) | 较新，仍在完善 |

- 跨平台 → **Qt**
- Windows 稳定项目 → **WPF**
- Windows 新项目 + 最新 UI → **WinUI3**

---

## 学习路径

| 路线 | 顺序 | 适合 |
|------|------|------|
| **Web 全栈** | 前端基础 → 后端基础 → 前后端协作 | 想快速上手、做产品 |
| **桌面开发** | 选 Qt / WPF / WinUI3 其一 | 想做原生工具软件 |
