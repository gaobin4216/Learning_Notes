# 前后端协作

> **一句话：前端负责展示，后端负责数据，两者通过接口(API)配合。**

---

## 一、秒懂前后端

| 餐厅 | 对应 | 干什么 |
|------|------|--------|
| 前厅（门面、菜单、服务员） | **前端** | 顾客看得见，负责展示和点餐 |
| 后厨（厨师、仓库） | **后端** | 顾客看不见，负责做菜、存食材 |

---

## 二、前后端怎么配合（以登录为例）

```
用户           前端（浏览器）         后端（服务器）         数据库
 │  1.输入账号密码  │                    │                    │
 │───点击登录─────>│  2.发送请求         │                    │
 │                 │──────POST──────────>│  3.查询             │
 │                 │                     │──────查库──────────>│
 │                 │                     │<─────结果──────────│
 │                 │  5.返回结果         │                    │
 │                 │<──────响应──────────│                    │
 │  6.显示成功     │                    │                    │
 │<──弹出提示──────│                    │                    │
```

> **接口（API）**：前后端之间的"传话通道"。前端按约定格式发数据、收数据，不用关心后端怎么写的。

## 三、输入一个链接，背后发生了什么

```
https://www.baidu.com/s?wd=hello
│       │           │   │
│       │           │   └── 参数：告诉后端搜什么
│       │           └────── 路径：要哪个页面
│       └────────────────── 域名：服务器叫什么
└────────────────────────── 协议：怎么通信
```

**完整流程：**

1. **DNS 解析**：浏览器问 DNS 服务器"www.baidu.com 的 IP 是多少？" → 拿到 IP 地址
2. **发请求**：浏览器拿着 IP 向服务器发 HTTP 请求："给我首页内容"
3. **后端处理**：服务器收到请求 → 后端程序运行 → 查数据库 → 生成页面
4. **浏览器渲染**：拿到 HTML/CSS/JS → 解析骨架 → 上色排版 → 加载交互 → 页面显示

**一句话**：浏览器（前端载体）去服务器（后端）要代码 → 拿回来渲染成页面。

**为什么有的网站快有的慢？**
- 服务器距离远（跨国慢）
- 服务器性能差
- 页面内容多（图片、代码量大）
- 网速慢
- 后端逻辑复杂（查很多数据、做很多计算）

## 四、运行形态对比

不是所有程序都分前后端：

| 运行形式 | 分前后端？ | 要联网？ | 举例 |
|----------|-----------|---------|------|
| 网页网站 | 标准前后端分离 | 基本都要 | 淘宝、B站 |
| 纯本地软件 | 不分 | 断网可用 | 记事本、单机游戏 |
| 联网软件 | 客户端+服务端 | 必须联网 | 微信、LOL |
| Python 脚本 | 不分 | 按需 | 自动化工具 |

**简单记**：全在一台电脑跑 → 没有前后端之分。代码分散在本地 + 远程服务器 → 就有前后端。

## 五、全栈

**全栈 = 前端 + 后端都会，能独立完成一整个项目。**

| 角色 | 偏什么 |
|------|--------|
| 前端工程师 | 视觉、体验、界面 |
| 后端工程师 | 逻辑、数据、服务器 |
| 全栈工程师 | 两边都行，独立开发者常见 |

### 全栈学习路线（6-12 个月入门）

```
1-2 月：前端基础（HTML + CSS + JS）
  3 月：前端框架（Vue 或 React）
  4 月：后端语言 + 数据库（Python + MySQL）
  5 月：后端框架 + 写接口（Flask/Django）
  6 月：前后端联调（前端调后端接口，打通流程）
7-8 月：独立做一个全栈项目
```

**练手项目推荐**：个人博客 → Todo 应用 → 在线聊天室 → 仿电商网站

## 六、速查

| 概念 | 一句话 |
|------|--------|
| **前端** | 用户看到并操作的界面，跑在浏览器里 |
| **后端** | 背后处理数据和逻辑的服务器程序 |
| **API** | 前后端之间传数据的通道 |
| **数据库** | 后端的"账本" |
| **全栈** | 前端 + 后端都会的人 |
