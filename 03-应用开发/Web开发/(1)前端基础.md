# 前端基础

> **一句话：用户能直接看到、点击、滑动的所有界面。**

---

## 一、前端是什么

前端运行在浏览器里，负责：
- 页面长什么样（布局、颜色、字体）
- 点按钮会怎样（弹窗、跳转、动画）
- 表单输入、图片展示、下拉刷新等一切交互

---

## 二、为什么偏偏用 HTML + CSS + JS？

浏览器（Chrome、Edge、Firefox）是执行网页的"操作系统"，内部只认识这三种语言：

| 语言 | 浏览器怎么理解它 | 为什么不能换成别的 |
|------|------------------|-------------------|
| **HTML** | 内核有 HTML 解析器，把标签翻译成页面元素 | 全球统一标准（W3C 规范） |
| **CSS** | 内核有 CSS 引擎，计算元素的颜色、位置、大小 | 所有浏览器按同一套规则渲染 |
| **JavaScript** | 内核有 JS 引擎（Chrome 的叫 V8），执行交互逻辑 | 浏览器唯一原生支持的编程语言 |

**通俗比喻**：浏览器就像一个只收"美元"的自动售货机——HTML/CSS/JS 是它唯一接受的"货币"，这是历史形成的行业标准，不是谁规定的"最佳语言"。

> 为什么不用 Python 写网页？因为浏览器不认识 Python。但 Python 可以写后端，后端跑在服务器上，不需要浏览器执行。

## 三、三大核心技术

| 技术 | 比喻 | 干什么 | 举例 |
|------|------|--------|------|
| **HTML** | 房子骨架 | 页面上有什么——标题、按钮、图片 | `<h1>标题</h1>` |
| **CSS** | 装修 | 好不好看——颜色、大小、布局 | `color: red;` |
| **JavaScript** | 智能开关 | 能不能动——点击、弹窗、动画 | `onclick → alert()` |

> **TypeScript (TS)** = JS 的升级版，加了类型检查（提前发现错误），大厂主流。

## 四、前端框架

框架 = 精装房模板，不用自己一块砖一块砖砌。

| 框架 | 特点 | 谁在用 |
|------|------|--------|
| **Vue** | 国内最火，入门简单，文档友好 | 小米、饿了么、B站 |
| **React** | 国外最火，生态庞大，灵活度高 | Meta、阿里、字节 |
| **UniApp** | 一份代码，同时生成 APP + 小程序 + 网页 | 中小企业、跨平台需求 |

## 五、代码示例

```html
<!-- HTML：结构 -->
<button>点我登录</button>

<!-- CSS：样式 -->
<style>
  button {
    background-color: blue;
    color: white;
    padding: 10px 20px;
    border-radius: 5px;
  }
</style>

<!-- JavaScript：交互 -->
<script>
  document.querySelector('button').onclick = function() {
    alert('登录成功！');
  }
</script>
```

HTML 写"有什么"，CSS 写"长什么样"，JS 写"点了会怎样"。

## 六、网页 vs 本地应用

| | 浏览器网页 | 本地应用（.exe） |
|---|---|---|
| **代码跑在哪** | 浏览器沙箱里 | 直接跑在操作系统上 |
| **用什么语言写** | HTML + CSS + JS | C++、C#、Python……都行 |
| **安装** | 不需要，打开浏览器就能用 | 需要下载安装 |
| **更新** | 刷新页面就是最新版 | 需要重新下载或自动更新 |
| **跨平台** | 有浏览器的设备都能用 | 需要各平台单独开发 |
| **权限** | 受浏览器限制 | 可访问文件、硬件、系统资源 |
| **性能** | 受浏览器约束 | 直接调用系统资源 |

**为什么网页这么多？** 不用安装、跨平台、更新方便、开发成本低。

**为什么还要做本地软件？** 需要调用硬件、高性能、离线使用、系统级权限。

## 七、学习路线

| 阶段 | 学什么 | 时间 | 效果 |
|------|--------|------|------|
| 1 | HTML 基础 | 1-2 周 | 能搭建简单页面结构 |
| 2 | CSS 基础 | 2-3 周 | 掌握盒模型、Flex 布局 |
| 3 | JavaScript | 4-6 周 | 能写交互逻辑，理解 DOM |
| 4 | Vue 或 React | 4-6 周 | 能开发单页面应用 |
| 5 | 做项目练手 | 4-8 周 | 完成一个完整个人项目 |

**建议**：前三阶段是基本功，不要跳过。每学一个知识点就动手写代码，光看不练等于没学。
