# 后端基础

> **一句话：背后处理数据和逻辑的服务器程序，用户看不见。**

---

## 一、后端在干什么

```
                        ┌─────────────────────────────────┐
                        │         后端服务器               │
  前端发来请求 ─────────>│  1. 接收请求                     │
                        │  2. 验证身份                     │
                        │  3. 处理业务                     │
                        │  4. 读写数据库                   │
                        │  5. 返回结果                     │───> 返回给前端
                        └─────────────────────────────────┘
```

---

## 二、五大核心职责

### 1. 接收和响应请求

前端发来"我要登录"的请求，后端接收、处理、返回结果。

### 2. 业务逻辑处理

| 业务场景 | 后端要做的事 |
|----------|-------------|
| 用户注册 | 检查用户名是否存在 → 加密密码 → 存数据库 → 发验证邮件 |
| 电商下单 | 验证库存 → 计算价格 → 扣减库存 → 生成订单 → 通知卖家 |
| 转账 | 验证余额 → 扣转出方 → 加转入方 → 记流水 → 发通知 |

### 3. 数据库操作

| 类型 | 代表产品 | 适合存什么 |
|------|---------|-----------|
| **关系型** | MySQL、PostgreSQL | 用户、订单、商品（固定表格结构） |
| **缓存** | Redis | 热门数据、验证码、登录状态（极快读写） |
| **文档型** | MongoDB | 日志、用户行为、非结构化数据 |

### 4. 接口（API）服务

接口 = 前后端之间的"传话协议"。

```
前端 → POST /api/login  {username: "zhangsan", password: "123456"} → 后端
前端 ← {code: 200, message: "登录成功", token: "abc123"}          ← 后端
```

- **地址**：前端往哪里发请求（如 `/api/login`）
- **格式**：数据长什么样（通常是 JSON）
- **方法**：GET（查询）、POST（提交）、PUT（修改）、DELETE（删除）

### 5. 安全与权限

| 措施 | 作用 | 举例 |
|------|------|------|
| 身份认证 | 确认"你是谁" | 登录、Token 验证 |
| 权限控制 | 确认"你能干什么" | 普通用户不能删帖子 |
| 数据加密 | 保护敏感信息 | 密码存哈希值，不存明文 |
| 防攻击 | 防止恶意请求 | 限频（防刷）、过滤输入（防注入） |

## 三、主流技术栈

| 语言 | 框架 | 特点 | 适合场景 |
|------|------|------|---------|
| **Python** | Flask、Django、FastAPI | 语法简洁，上手快 | 中小型项目、AI 服务 |
| **Java** | Spring Boot | 稳定、生态成熟 | 大型企业系统、金融、电商 |
| **Go** | Gin、Echo | 性能极高，并发强 | 高并发、微服务、云原生 |
| **Node.js** | Express、Nest.js | 前后端都用 JS | 全栈开发、实时应用 |
| **C#** | ASP.NET | 微软生态 | 企业内部系统、游戏后端 |

## 四、Flask 示例

```python
from flask import Flask, request, jsonify

app = Flask(__name__)

@app.route('/api/login', methods=['POST'])
def login():
    username = request.json.get('username')
    password = request.json.get('password')

    if username == 'zhangsan' and password == '123456':
        return jsonify({'code': 200, 'message': '登录成功'})
    else:
        return jsonify({'code': 401, 'message': '账号或密码错误'})

if __name__ == '__main__':
    app.run(port=5000)
```

启动服务器 → 等待前端发 `POST /api/login` → 验证 → 返回结果。

## 五、学习路线

| 阶段 | 学什么 | 时间 | 效果 |
|------|--------|------|------|
| 1 | 选语言（推荐 Python） | 3-4 周 | 掌握基础语法 |
| 2 | 数据库（MySQL） | 2-3 周 | 能建表、增删改查 |
| 3 | Web 框架 | 3-4 周 | 能用 Flask/Django 写接口 |
| 4 | 接口设计 | 2-3 周 | 理解 RESTful 风格 |
| 5 | 部署 | 2-3 周 | 能把服务部署到服务器上 |
| 6 | 做项目 | 4-8 周 | 完成一个完整后端项目 |

**建议**：Python 入门门槛最低；目标进大厂可以考虑 Java。数据库是基本功，SQL 必须熟练。
