# 前后端入门指南（零基础友好）

> 一句话搞懂：**前端 = 你看到的界面，后端 = 界面背后干活的服务器。**

---

## 一、餐厅比喻：秒懂前后端

把网站 / APP 比作一家餐厅：

| 餐厅 | 对应 | 干什么 |
|------|------|--------|
| 前厅（门面、菜单、餐桌、服务员） | **前端** | 顾客看得见、摸得着，负责展示和点餐 |
| 后厨（厨师、食材仓库、收银台） | **后端** | 顾客看不见，负责做菜、算账、存食材 |

你作为顾客（用户），只和前厅打交道，但真正把菜做出来的是后厨。

---

## 二、前端是什么

**一句话：用户能直接看到、点击、滑动的所有界面。**

它运行在你的浏览器、手机、平板上。

### 前端负责什么

- 页面长什么样（布局、颜色、字体）
- 点按钮会怎样（弹窗、跳转、动画）
- 表单输入、图片展示、下拉刷新等一切交互

---

### 为什么偏偏用 HTML + CSS + JS？

你可能会问：世界上编程语言那么多（Python、C++、Java……），为什么网页非得用这三样？

**答案：这是浏览器规定的。**

浏览器（Chrome、Edge、Firefox、Safari）是执行网页的"操作系统"。它内部只认识这三种语言：

| 语言 | 浏览器怎么理解它 | 为什么不能换成别的 |
|------|------------------|-------------------|
| **HTML** | 浏览器内核专门有一个 HTML 解析器，把标签翻译成页面元素 | 全球统一标准（W3C 规范），所有浏览器都按同一套规则解析 |
| **CSS** | 浏览器内核有 CSS 引擎，专门计算每个元素的颜色、位置、大小 | 如果每家浏览器各用各的样式语言，网页在不同浏览器上就乱了 |
| **JavaScript** | 浏览器内核有 JS 引擎（Chrome 的叫 V8），专门执行交互逻辑 | JS 是浏览器唯一原生支持的编程语言，别的语言浏览器不认 |

**通俗比喻：**

浏览器就像一个只收"美元"的自动售货机：
- 你塞人民币、日元、欧元，它都不认
- 只有 HTML / CSS / JS 这三种"货币"它才收
- 这不是谁规定的"最佳语言"，而是历史形成的行业标准——全世界的浏览器都按这套标准造，所以全世界的前端都用这三种语言

> 那为什么不用 Python 写网页？因为浏览器不认识 Python。但你完全可以用 Python 写后端，后端跑在服务器上，不需要浏览器来执行。

### 前端三大核心技术详解

| 技术 | 比喻 | 干什么 | 举例 |
|------|------|--------|------|
| **HTML** | 房子骨架 | 决定页面上有什么——标题、按钮、图片、输入框 | `<h1>标题</h1>`、`<button>按钮</button>` |
| **CSS** | 装修 | 决定好不好看——颜色、大小、位置、布局 | `color: red;`、`font-size: 16px;` |
| **JavaScript (JS)** | 智能开关 | 决定能不能动——点击、滑动、弹窗、轮播图 | 点击按钮弹出提示、表单验证、动画效果 |

> **TypeScript (TS)** = JS 的升级版，加了"类型检查"（提前发现错误），大厂主流。

### 前端框架（不用从零写，效率翻倍）

框架 = 精装房模板，不用自己一块砖一块砖砌。

| 框架 | 特点 | 谁在用 |
|------|------|--------|
| **Vue** | 国内最火，入门简单，文档友好 | 小米、饿了么、B站、个人开发者 |
| **React** | 国外最火，生态庞大，灵活度高 | Meta、阿里、腾讯、字节跳动 |
| **UniApp** | 一份代码，同时生成 APP + 小程序 + 网页 | 中小企业、跨平台需求 |

### 前端代码长什么样

```html
<!-- HTML：结构——页面上有什么 -->
<button>点我登录</button>

<!-- CSS：样式——长什么样 -->
<style>
  button {
    background-color: blue;   /* 背景色蓝色 */
    color: white;             /* 文字白色 */
    padding: 10px 20px;       /* 内边距 */
    border-radius: 5px;       /* 圆角 */
    font-size: 16px;          /* 字号 */
  }
</style>

<!-- JavaScript：交互——点了会怎样 -->
<script>
  document.querySelector('button').onclick = function() {
    alert('登录成功！');
  }
</script>
```

HTML 写"有什么"，CSS 写"长什么样"，JS 写"点了会怎样"。三者配合，才构成一个完整的网页。

## 三、浏览器网页 vs 本地应用（搞清楚区别）

很多人搞不清"网页"和"软件"到底有什么区别，这里讲透。

### 核心区别：谁来执行代码

| 对比项 | 浏览器网页 | 本地应用（.exe） |
|--------|-----------|-----------------|
| **代码跑在哪** | 跑在浏览器里面，浏览器是"中间人" | 直接跑在操作系统上，没有中间人 |
| **用什么语言写** | HTML + CSS + JS（浏览器只认这三样） | C++、C#、Java、Python、Go……什么语言都行 |
| **怎么打开** | 输入网址，或者点链接 | 双击图标，或者从开始菜单启动 |
| **安装需要吗** | 不需要安装，打开浏览器就能用 | 需要下载安装包，装到电脑上 |
| **更新麻烦吗** | 开发者改了代码，你刷新页面就是最新版 | 需要你重新下载安装新版（或者软件内自动更新） |
| **跨平台** | 任何有浏览器的设备都能用（电脑、手机、平板） | Windows 的 .exe 在 Mac 上跑不了，需要单独开发 |
| **权限** | 受浏览器限制，不能随便访问你的文件系统 | 可以访问电脑上的文件、硬件、系统资源 |
| **性能** | 受浏览器约束，跑大型计算会慢 | 直接调用系统资源，性能更好 |

### 三种形态对比

```
┌─────────────────────────────────────────────────────────┐
│  浏览器网页                                              │
│  ┌─────────────────────────────────────────────────┐    │
│  │  浏览器（Chrome / Edge）                         │    │
│  │  ┌─────────────────────────────────────────┐    │    │
│  │  │  你的网页代码（HTML + CSS + JS）          │    │    │
│  │  │  跑在浏览器提供的"沙箱"环境里             │    │    │
│  │  └─────────────────────────────────────────┘    │    │
│  └─────────────────────────────────────────────────┘    │
│  操作系统（Windows / Mac / Linux）                       │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│  本地 .exe 软件                                          │
│  ┌─────────────────────────────────────────────────┐    │
│  │  你的软件代码（C++ / C# / Python 等）             │    │
│  │  直接跑在操作系统上，可以调用系统资源              │    │
│  └─────────────────────────────────────────────────┘    │
│  操作系统（Windows / Mac / Linux）                       │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│  联网 .exe 软件（微信、网游）                             │
│  ┌─────────────────────────────────────────────────┐    │
│  │  客户端（本地界面 + 基础逻辑）                     │    │
│  │  跑在你电脑上，类似"不用浏览器的前端"              │    │
│  └──────────────────┬──────────────────────────────┘    │
│                     │ 网络通信                           │
│  ┌──────────────────▼──────────────────────────────┐    │
│  │  远程服务器（后端）                               │    │
│  │  处理数据、存聊天记录、算游戏逻辑                  │    │
│  └─────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────┘
```

### 为什么网页这么多？为什么不做成本地软件？

| 原因 | 解释 |
|------|------|
| **不用安装** | 用户打开浏览器就能用，降低使用门槛 |
| **跨平台** | 一套代码在 Windows、Mac、手机上都能跑 |
| **更新方便** | 开发者改了代码，用户刷新就是新版，不用重新下载 |
| **开发成本低** | 一套前端代码适配所有设备，不用给每个平台单独开发 |

那为什么还要做本地软件？
- 需要调用硬件（摄像头、显卡、蓝牙）
- 需要高性能（大型游戏、视频剪辑）
- 需要离线使用（断网也能用）
- 需要系统级权限（读写本地文件、开机自启）

## 四、后端深入详解

### 后端到底在干什么？

后端就像一家公司的"后台部门"，所有核心业务都在这里处理：

```
                        ┌─────────────────────────────────┐
                        │         后端服务器               │
                        │                                 │
  前端发来请求 ─────────>│  1. 接收请求（收快递）            │
                        │  2. 验证身份（查身份证）          │
                        │  3. 处理业务（干活）              │
                        │  4. 读写数据库（记账）            │
                        │  5. 返回结果（回快递）            │───> 返回给前端
                        │                                 │
                        └─────────────────────────────────┘
```

### 后端五大核心职责

#### 1. 接收和响应请求

前端发来"我要登录"的请求，后端接收、处理、返回结果。就像快递收件和发件。

#### 2. 业务逻辑处理

这是后端最核心的工作，举几个例子：

| 业务场景 | 后端要做的事 |
|----------|-------------|
| 用户注册 | 检查用户名是否已存在 → 加密密码 → 存入数据库 → 发送验证邮件 |
| 电商下单 | 验证库存 → 计算价格（含优惠券、满减）→ 扣减库存 → 生成订单 → 通知卖家 |
| 发布动态 | 检查内容是否违规 → 存入数据库 → 推送给粉丝 → 更新动态流 |
| 转账 | 验证余额 → 扣除转出方 → 增加转入方 → 记录流水 → 发送通知 |

#### 3. 数据库操作

数据库 = 后端的"账本"，所有数据都存在这里。

| 数据库类型 | 代表产品 | 适合存什么 | 比喻 |
|-----------|---------|-----------|------|
| **关系型数据库** | MySQL、PostgreSQL | 用户信息、订单、商品（有固定表格结构） | Excel 表格，一行一条记录 |
| **缓存数据库** | Redis | 热门数据、验证码、登录状态（要求极快读写） | 便签纸，随手记随手拿 |
| **文档型数据库** | MongoDB | 日志、用户行为、非结构化数据 | 文件柜，存各种格式的文档 |

#### 4. 接口（API）服务

接口 = 前后端之间的"传话协议"。

```
前端：我要登录，账号是 zhangsan，密码是 123456
      ──── POST /api/login  {username: "zhangsan", password: "123456"} ────>

后端：查了一下，密码正确
      <──── {code: 200, message: "登录成功", token: "abc123"} ────────────

前端：收到，显示"登录成功"
```

接口规定了：
- **地址**：前端往哪里发请求（比如 `/api/login`）
- **格式**：数据长什么样（通常是 JSON 格式）
- **方法**：什么操作（GET = 查询，POST = 提交，PUT = 修改，DELETE = 删除）

#### 5. 安全与权限

| 安全措施 | 作用 | 举例 |
|---------|------|------|
| 身份认证 | 确认"你是谁" | 登录、Token 验证 |
| 权限控制 | 确认"你能干什么" | 普通用户不能删帖子，管理员可以 |
| 数据加密 | 保护敏感信息 | 密码不存明文，存加密后的哈希值 |
| 防攻击 | 防止恶意请求 | 限制频繁请求（防刷）、过滤恶意输入（防注入） |

### 后端主流技术详解

| 类别 | 技术 | 特点 | 适合场景 |
|------|------|------|---------|
| **Python** | Flask（轻量）、Django（全栈）、FastAPI（高性能） | 语法简洁，上手快，AI/数据领域首选 | 中小型项目、快速原型、AI 服务 |
| **Java** | Spring Boot | 稳定、生态成熟、企业级首选 | 大型企业系统、金融、电商 |
| **Go** | Gin、Echo | 性能极高，并发能力强 | 高并发服务、微服务、云原生 |
| **Node.js** | Express、Nest.js | 前后端都用 JS，一套语言走天下 | 全栈开发、实时应用（聊天室） |
| **C#** | ASP.NET | 微软生态，和 Windows 深度集成 | 企业内部系统、游戏后端 |
| **PHP** | Laravel、ThinkPHP | 部署简单，老牌 Web 语言 | 中小型网站、CMS 系统 |

### 后端代码长什么样（Python Flask 示例）

```python
from flask import Flask, request, jsonify

app = Flask(__name__)

# 定义一个登录接口：POST /api/login
@app.route('/api/login', methods=['POST'])
def login():
    # 1. 获取前端发来的数据
    username = request.json.get('username')
    password = request.json.get('password')

    # 2. 业务逻辑：查询数据库，验证密码
    if username == 'zhangsan' and password == '123456':
        # 3. 验证通过，返回成功
        return jsonify({'code': 200, 'message': '登录成功'})
    else:
        # 4. 验证失败，返回错误
        return jsonify({'code': 401, 'message': '账号或密码错误'})

# 启动服务器，监听 5000 端口
if __name__ == '__main__':
    app.run(port=5000)
```

这段代码做的事：
- 启动一个服务器，等待前端发请求
- 当前端发来 `POST /api/login` 请求时，取出账号密码
- 验证后返回"登录成功"或"账号或密码错误"
- 前端收到结果后，显示对应的提示

## 五、前后端怎么配合（以登录为例）

```
用户（你）          前端（浏览器）           后端（服务器）          数据库
   |                    |                      |                    |
   |  1.输入账号密码     |                      |                    |
   |---点击"登录"------>|                      |                    |
   |                    |  2.发送账号密码        |                    |
   |                    |------请求接口--------->|                    |
   |                    |                      |  3.查询账号密码      |
   |                    |                      |--------查询-------->|
   |                    |                      |  4.返回查询结果      |
   |                    |                      |<--------结果--------|
   |                    |  5.返回"登录成功"      |                    |
   |                    |<------响应结果---------|                    |
   |  6.显示"登录成功"   |                      |                    |
   |<--弹出提示---------|                      |                    |
```

> **接口（API）**：前后端之间的"传话通道"。前端不用关心后端怎么写的，只要按约定的格式发数据、收数据就行。

## 六、输入一个链接，浏览器到底做了什么？

你每天都在浏览器地址栏输入链接、点链接打开网页，但背后到底发生了什么？这和前后端有什么关系？

### 一个链接长什么样

```
https://www.baidu.com/s?wd=hello
│       │           │   │
│       │           │   └── 参数：告诉后端你要搜什么（hello）
│       │           └────── 路径：告诉后端你要哪个页面（搜索页）
│       └────────────────── 域名：这台服务器叫什么名字
└────────────────────────── 协议：用什么方式通信（https = 加密通信）
```

### 从输入链接到页面展示的完整过程

```
你输入 www.baidu.com 并按回车
         │
         ▼
┌──────────────────────────────────────────────────────────────────┐
│ 第 1 步：浏览器问 DNS："www.baidu.com 的服务器在哪？"            │
│                                                                  │
│   DNS = 互联网的"电话簿"                                        │
│   域名（www.baidu.com）→ IP 地址（110.242.68.66）               │
│   就像你打电话给 114 查号台："百度公司的电话号码是多少？"         │
│   DNS 服务器查到后告诉浏览器：去 110.242.68.66 这个地址           │
└──────────────────────────┬───────────────────────────────────────┘
                           │
                           ▼
┌──────────────────────────────────────────────────────────────────┐
│ 第 2 步：浏览器向服务器发起请求                                    │
│                                                                  │
│   浏览器拿到 IP 地址后，向那台服务器发一个 HTTP 请求：            │
│   "你好，我要访问首页，请把页面内容发给我"                        │
│   就像你打电话给百度公司："请把你们的首页发给我看看"              │
└──────────────────────────┬───────────────────────────────────────┘
                           │
                           ▼
┌──────────────────────────────────────────────────────────────────┐
│ 第 3 步：后端服务器处理请求                                        │
│                                                                  │
│   服务器收到请求后，后端程序开始工作：                             │
│   - 如果是静态页面：直接把 HTML 文件发回去                        │
│   - 如果是动态页面：后端程序运行 → 查数据库 → 生成 HTML → 发回去  │
│   - 如果是搜索请求：后端拿到搜索词 → 查数据库 → 返回搜索结果      │
│                                                                  │
│   就像百度公司的前台收到你的电话，转给相关部门处理                 │
└──────────────────────────┬───────────────────────────────────────┘
                           │
                           ▼
┌──────────────────────────────────────────────────────────────────┐
│ 第 4 步：浏览器收到响应，开始渲染页面                              │
│                                                                  │
│   服务器返回的内容通常包括：                                       │
│   - HTML 文件（骨架）                                             │
│   - CSS 文件（样式）                                              │
│   - JS 文件（交互）                                               │
│   - 图片、字体等资源                                              │
│                                                                  │
│   浏览器拿到这些文件后：                                           │
│   1. 解析 HTML → 搭建页面骨架                                     │
│   2. 解析 CSS → 给骨架上色、排版                                  │
│   3. 解析 JS → 让页面能点击、能交互                               │
│   4. 加载图片等资源 → 页面完整显示                                 │
│                                                                  │
│   这就是你看到的网页！                                             │
└──────────────────────────────────────────────────────────────────┘
```

### 用大白话总结整个过程

```
你输入链接
    │
    ▼
浏览器问 DNS："这个网站的服务器地址是多少？"  ──→  DNS 返回 IP 地址
    │
    ▼
浏览器拿着 IP 地址，向服务器发请求："给我页面内容"  ──→  服务器收到请求
    │
    ▼
后端程序处理请求（查数据库、算数据、生成页面）  ──→  把 HTML/CSS/JS 发回来
    │
    ▼
浏览器拿到代码，渲染成你看到的漂亮页面
```

### 这和前后端有什么关系？

| 环节 | 谁在干活 | 前端还是后端 |
|------|---------|-------------|
| 输入链接、发起请求 | 浏览器 | 前端的"载体" |
| DNS 解析域名 | DNS 服务器 | 基础设施，不属于前后端 |
| 接收请求、处理业务 | 远程服务器上的程序 | **后端** |
| 查数据库、返回数据 | 后端程序 + 数据库 | **后端** |
| 拿到代码、渲染页面 | 浏览器 | **前端** |
| 你看到页面、点击交互 | 浏览器里的网页 | **前端** |

**一句话：你输入链接 → 浏览器（前端载体）去服务器（后端）要代码 → 拿回来渲染成页面（前端展示）。整个过程就是前后端的一次完整配合。**

### 为什么有的链接打开快，有的慢？

| 影响因素 | 说明 |
|---------|------|
| **服务器距离** | 服务器在国外比在国内慢（数据要跨洋传输） |
| **服务器性能** | 服务器配置高、处理快，响应就快 |
| **网页内容多少** | 图片多、代码多的页面加载慢 |
| **你的网速** | 网速慢，下载页面代码的时间就长 |
| **后端逻辑复杂度** | 需要查很多数据、做很多计算的页面响应慢 |

## 七、运行形态对比（不是所有程序都分前后端）

| 运行形式 | 运行位置 | 分前后端？ | 要联网？ | 举例 |
|----------|----------|-----------|---------|------|
| 网页网站 | 前端在浏览器，后端在服务器 | 标准前后端分离 | 基本都要 | 淘宝、B站网页版 |
| 纯本地软件（.exe） | 全在本机 | 不分 | 断网可用 | 记事本、单机游戏 |
| 联网软件（微信、网游） | 界面在本机，数据在服务器 | 客户端+服务端 | 必须联网 | 微信电脑版、LOL |
| Python 脚本 | 全在本机 | 不分 | 按需 | 自动化工具、数据处理 |
| Python 后端服务 | 远程服务器 | 纯后端 | 对外服务 | Flask / Django 写的接口 |

**简单记：**
- 全在一台电脑跑 → 没有前后端之分
- 代码分散在本地 + 远程服务器 → 就有前端 / 后端的概念

## 八、全栈是什么

**全栈 = 前端 + 后端都会，能独立完成一整个项目。**

- 前端工程师：偏视觉、体验、和界面打交道
- 后端工程师：偏逻辑、数据、和服务器数据库打交道
- 全栈工程师：两边都行，独立开发者常见

## 九、新手学习路线（详细版）

### 路线一：学前端（约 3-6 个月入门）

| 阶段 | 学什么 | 学多久 | 要达到的效果 | 推荐资源 |
|------|--------|--------|-------------|---------|
| **第 1 阶段** | HTML 基础 | 1-2 周 | 能用标签搭建出一个简单的页面结构 | W3School、MDN 文档 |
| **第 2 阶段** | CSS 基础 | 2-3 周 | 能让页面变好看，掌握盒模型、Flex 布局 | CSS-Tricks、B站免费教程 |
| **第 3 阶段** | JavaScript 基础 | 4-6 周 | 能写交互逻辑，理解变量、函数、DOM 操作 | 《JavaScript 高级程序设计》、freeCodeCamp |
| **第 4 阶段** | Vue 或 React 框架 | 4-6 周 | 能用框架开发单页面应用 | Vue 官方教程（新手推荐）、React 官方文档 |
| **第 5 阶段** | 做项目练手 | 4-8 周 | 做一个完整的个人项目（比如 Todo 应用、博客、仿电商页面） | GitHub 上搜"前端练手项目" |
| **第 6 阶段** | 进阶 | 持续学习 | TypeScript、Webpack/Vite 构建工具、性能优化、响应式布局 | 工作中学习、技术博客 |

**学习建议：**
- 前三阶段是基本功，不要跳过，地基不牢后面会很痛苦
- 每学一个知识点就动手写代码，光看不练等于没学
- 第 5 阶段的项目非常重要，面试全靠项目经验

### 路线二：学后端（约 3-6 个月入门）

| 阶段 | 学什么 | 学多久 | 要达到的效果 | 推荐资源 |
|------|--------|--------|-------------|---------|
| **第 1 阶段** | 选一门语言（推荐 Python） | 3-4 周 | 掌握基础语法、变量、函数、循环、条件判断 | 《Python 编程：从入门到实践》、菜鸟教程 |
| **第 2 阶段** | 学数据库（MySQL） | 2-3 周 | 能建表、增删改查、写简单 SQL 语句 | MySQL 官方教程、LeetCode SQL 练习 |
| **第 3 阶段** | 学 Web 框架 | 3-4 周 | 能用框架写接口（Flask 或 Django） | Flask 官方文档、B站搜"Flask 教程" |
| **第 4 阶段** | 学接口设计 | 2-3 周 | 理解 RESTful 风格、能设计合理的 API | 搜"RESTful API 设计规范" |
| **第 5 阶段** | 学部署 | 2-3 周 | 能把后端服务部署到服务器上，让别人能访问 | 学 Linux 基本命令、Nginx 配置、云服务器（阿里云/腾讯云学生机） |
| **第 6 阶段** | 做项目练手 | 4-8 周 | 做一个完整的后端项目（比如博客 API、用户管理系统） | 从零设计数据库 → 写接口 → 部署上线 |

**学习建议：**
- Python 入门门槛最低，适合新手；如果目标是进大厂，可以考虑 Java
- 数据库是后端的基本功，SQL 必须熟练
- 第 5 阶段的部署很重要，很多新手学完代码却不知道怎么上线

### 路线三：学全栈（约 6-12 个月入门）

```
第 1-2 个月：前端基础（HTML + CSS + JS）
第 3 个月：  前端框架（Vue 或 React）
第 4 个月：  后端语言 + 数据库（Python + MySQL）
第 5 个月：  后端框架 + 写接口（Flask 或 Django）
第 6 个月：  前后端联调（前端调后端接口，打通完整流程）
第 7-8 个月：独立做一个全栈项目（前端 + 后端 + 数据库 + 部署上线）
```

**全栈项目推荐（从易到难）：**
1. 个人博客系统（注册登录、发布文章、评论）
2. Todo 待办应用（增删改查、状态管理）
3. 在线聊天室（WebSocket 实时通信）
4. 仿电商网站（商品展示、购物车、下单）

## 十、常见误区

| 误区 | 正确理解 |
|------|---------|
| 前端就是"美工" | 前端不只是切图，还涉及性能优化、交互逻辑、工程化 |
| 后端比前端难 | 各有各的难点，不存在谁比谁"高级" |
| Python 主要做前端 | Python 主要做后端，前端主流是 JS/TS |
| 学了 HTML 就会做网页 | HTML 只是骨架，还需要 CSS + JS + 框架配合 |
| 前后端必须分两个人做 | 小项目一个人可以全做（全栈），大项目才需要分工 |
| 网页和软件是一回事 | 网页跑在浏览器里，软件跑在操作系统上，底层机制完全不同 |
| 学编程一定要先学前端 | 看你的目标，后端、数据分析、AI 方向可以不学前端 |

## 十一、速查总结

| 概念 | 一句话解释 |
|------|-----------|
| **HTML** | 有什么（页面结构） |
| **CSS** | 好不好看（页面样式） |
| **JavaScript** | 能不能点（页面交互） |
| **前端** | 用户看到并操作的界面，跑在浏览器里 |
| **后端** | 背后处理数据和逻辑的服务器程序 |
| **API** | 前后端之间传数据的通道 |
| **数据库** | 后端的"账本"，所有数据存在这里 |
| **浏览器** | 执行网页代码的"中间人"，只认 HTML/CSS/JS |
| **全栈** | 前端 + 后端都会的人 |
| **框架** | 现成的代码模板，加速开发 |

> 前端负责"展示和交互"，后端负责"计算和存数据"，浏览器是两者之间的"舞台"。
