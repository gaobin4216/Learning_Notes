# 大模型基础概念

## 一、什么是大模型

**大模型**，全称**大规模预训练人工智能模型**，是依托海量数据训练、拥有巨量参数的深度学习模型。

- **参数规模大**：百亿、千亿级神经网络参数，模拟人脑神经元运算
- **预训练基底**：先用全网文本、图像、音视频等海量数据完成基础学习
- **通用能力强**：可对话、创作、推理、翻译、识图、写代码，适配多场景
- **深度学习驱动**：依靠算法自主理解、生成内容，具备泛化思考能力

> **大模型**：统称所有超大参数 AI 模型，包含语言、图像、多模态等全部类型。平时说的大模型，大多默认指 LLM（Large Language Model，大语言模型）。

---

## 二、基础概念

### 1. Token（词元）

**与 AI 大模型交互的基本单位**：当你向大模型提问时，输入会被拆分成一个个编码片段，每个编码片段就是一个 Token。

> **编码规则**：模型出厂时预设的固定码表，所有文字按统一规则切成 Token 并转为数字。不同模型的码表不同，不可通用。切词、编码、解码全程按既定规则机械执行，不会临时改动。

| 属性 | 说明 |
|------|------|
| **定义** | 模型处理文本的最小单位（字、词、词根、标点） |
| **换算** | 中文 ≈ 1.6–2 字 / Token；英文 ≈ 0.75 单词 / Token |
| **用途** | 计费 + 算力消耗（ChatGPT 按 Token 收费） |
| **示例** | "我爱吃苹果" → 我、爱吃、苹果 → 3 个 Token |

---

### 2. Context（上下文）

大模型每次处理任务时接收到的信息总和，相当于模型的**临时记忆体**，包含上下文信息及其他相关信息。

#### Context Window（上下文窗口）

Context 能容纳的最大 Token 数量。

---

### 3. Prompt（提示词）

向大模型提出的问题。好的 Prompt 能让模型更好地理解任务。

- **User Prompt**：用户提示词，即提问内容
- **System Prompt**：系统提示词，模型预置的规则说明

---

### 4. Tool（工具）

本质是一个**函数**，让大模型能感知并调用外部环境。例如查询天气时，模型本身无法推理出天气数据，需要调用天气查询 Tool 获取结果后返回给用户。

---

### 5. MCP（模型上下文协议）

**Model Context Protocol**，2024 年底由 Anthropic 推出的开放标准接口，统一大模型与外部世界的连接方式。

> MCP 不是模型，也不是功能，而是一套 **"通用插头标准"**，让大模型能安全、标准化、低成本地连接数据库、文件、API 和各种工具，是 AI Agent、长上下文、企业知识库的关键基础设施。

> **类比**：MCP = 大模型的 USB-C 接口，让任何工具 / 数据库 / 文件系统都能"即插即用"地连接到大模型。工具开发者只需按此标准开发，所有支持 MCP 的平台即可使用该工具。

---

### 6. Agent（智能体）

能**自主思考、规划、执行任务**的 AI 程序，不再是一问一答的模式。

- **思考规划**：拆分复杂任务
- **工具调用**：联网、查数据、运行程序、读写文件
- **循环执行**：完成后核对，未完成则继续

Agent 是智能体框架 / 程序，负责思考规划、拆解任务、调用工具、流转逻辑，可自由切换接入不同模型（Claude、GPT、通义、本地开源模型等）。一套 Agent 流程，换模型接口即可切换算力来源，框架本身不携带模型权重，仅做请求收发与任务调度。

---

### 7. Agent Skill（智能体技能）

提前编写好的、给 Agent 的说明文档，可理解为**按需调用的高级 System Prompt**，规定 Agent 做事的步骤和规则。

---

### 8. Harness（驾驭框架）

除了 LLM 本身之外，让 Agent 真正能干活的一切基础设施。

> **核心公式：Agent = LLM + Harness**

Harness 是除 LLM 本体外，支撑 Agent 完成生产任务的全部基础设施与工程体系的总和。它并非优化 Prompt 或升级模型，而是通过构建模型运行的环境、机制、规则与边界，实现 AI 智能的工程化落地。

---

## 三、AI 关键概念的关系

### 1. Agent 与 Harness

**Agent = Model + Harness**

Agent 是完整智能体，Harness 是其非模型的支撑与执行框架。

![Agent、Model、Harness 关系图](images/image-2.png)

| 概念 | 说明 | 类比 |
|------|------|------|
| **Model（模型）** | LLM 等基础模型，负责核心推理、理解与生成 | 赛车引擎（强动力但无法单独行驶） |
| **Harness（驾驭框架）** | 模型之外的所有工程化支撑，含编排、工具调用、记忆、状态管理、上下文组装、验证与安全边界 | 底盘、方向盘、刹车、导航、车身（让引擎成为可上路的车） |
| **Agent（智能体）** | 能感知环境、自主决策、调用工具、完成目标的完整行为主体 | 完整赛车（引擎 + 整车系统） |

---

### 2. Prompt、Context、Harness 的关系

| 概念 | 一句话理解 | 考试类比 |
|------|------------|----------|
| **Prompt** | 一句话怎么写模型才懂 | 这道题的题目 |
| **Context** | 给模型什么背景资料 | 考生桌上摆的所有东西（题目 + 草稿纸 + 参考资料 + 之前做过的题） |
| **Harness** | 模型接到任务后如何工作 | 考场规则 + 监考老师（决定什么时候发卷、翻页、交卷、继续下一轮） |

**执行流程：**

```
时间轴 ──────────────────────────────────────────────────────────▶
Harness 一直运行
  │
  ├─ 第1轮：组装 Context₁ → 塞入 Prompt₁ → 调用模型
  │         ← 模型输出 → 执行工具 → 得到结果
  │
  ├─ 第2轮：组装 Context₂（含上轮历史 + 工具结果）→ 塞入 Prompt₂
  │         ← 模型输出 → ...
  │
  └─ 第N轮：直到任务完成或达到终止条件
```

### 3. 智能体发展三阶段

Agent 的发展历史，是让模型从 **"会说话的文本生成器"** 进化为 **"能自主完成任务的智能体"** 的过程。

| 阶段 | 核心问题 | 说明 |
|------|----------|------|
| **阶段一：提示词工程（Prompt Engineering）** | "怎么跟模型说话" | 精心雕琢指令、Few-shot 示例、思维链（Chain-of-Thought）引导，但单次交互、无状态，更像手艺而非工程 |
| **阶段二：上下文工程（Context Engineering）** | "模型应该看到什么" | 开发者从"用户"转变为"Agent Builder"，系统性设计动态上下文系统（知识库、工具调用、记忆管理） |
| **阶段三：驾驭工程（Harness Engineering）** | "整个环境应该如何运作" | 主动为模型构建可运行、可约束、可验证的工作环境，即 Harness |
