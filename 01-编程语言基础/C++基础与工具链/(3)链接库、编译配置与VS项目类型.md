# 链接库、编译模式与 VS 项目类型

## 一、链接库基础

### 1. 什么是库

库是写好的、可复用的二进制代码，可以被操作系统载入内存执行。

程序从源码到可执行文件的流程：

```
源代码 → 预处理 → 编译 → 汇编 → 链接 → 可执行文件
```

库在**链接**阶段参与，分为两种方式：静态链接和动态链接。

### 2. 相关文件类型

| 文件类型 | Windows | Linux | 作用 | 需要时机 |
|----------|---------|-------|------|----------|
| 头文件 | `.h` | `.h` | 声明函数接口 | 编译时 |
| 静态库 | `.lib` | `.a` | 函数代码，编译时链接进目标程序 | 链接时 |
| 动态库 | `.dll` | `.so` | 函数可执行代码，运行时载入 | 运行时 |
| 导入库 | `.lib`（配合 .dll） | 无（Linux 不需要） | 地址符号表，告诉链接器函数在哪个动态库中 | 链接时 |

> Windows 的 .lib 有两种含义：**静态库的 .lib**（包含实际代码）和**动态库的导入库 .lib**（只包含地址符号表）。Linux 下静态库（.a）和动态库（.so）是完全独立的文件，不存在歧义。

### 3. 静态链接库

链接阶段将目标文件（.o/.obj）与引用的库一起打包到可执行文件中。静态库本质上是一组目标文件的集合。

组成：**.h + .lib**（函数代码在 .lib 中）

| 优点 | 缺点 |
|------|------|
| 运行时无外部依赖，移植方便 | 所有代码合入一个文件，体积大 |
| | 库更新后所有程序需重新编译 |

### 4. 动态链接库

程序编译时不链接动态库，运行时才载入。不同程序调用同一动态库时，内存中只需一份实例。

组成：**.h + .lib（导入库）+ .dll**（函数代码在 .dll 中）

| 优点 | 缺点 |
|------|------|
| 进程间资源共享，节省内存 | 需确保目标环境有对应 .dll |
| 库更新后只需替换 .dll | |
| 支持运行时动态加载（显式调用） | |

#### 显式调用 vs 隐式调用

- **隐式调用**：编译时指定库和查找路径，需要头文件。配置方式见下文"在 VS 中配置库"
- **显式调用**：运行时通过 `LoadLibrary` / `GetProcAddress` 动态加载，无需头文件。操作繁琐且危险，能隐式就不显式

## 二、在 VS 中使用库

### 1. 配置步骤

静态库和动态库的配置方式基本一致：

1. **附加包含目录**：`C/C++ → 常规 → 附加包含目录`，填入 .h 所在目录
2. **附加库目录**：`链接器 → 常规 → 附加库目录`，填入 .lib 所在目录
3. **附加依赖项**：`链接器 → 输入 → 附加依赖项`，填入 .lib 文件名

> 动态库额外一步：将 .dll 放到 .exe 所在目录，或配置系统环境变量。

> **包含目录 vs 附加包含目录**：前者在 `VC++ 目录` 下，是全局设置（基于环境变量）；后者是当前项目设置。推荐用附加目录。

### 2. Vcpkg 包管理器

vcpkg 是微软开发的跨平台 C/C++ 包管理器，最大优势是与 Visual Studio 无缝集成——自动配置头文件和库路径，无需手动设置。

- [vcpkg GitHub](https://github.com/Microsoft/vcpkg)
- [安装教程](https://www.cnblogs.com/hik-wxy/p/14744272.html)

## 三、Debug 与 Release

### 1. 两种编译模式

| 特性 | Debug | Release |
|------|-------|---------|
| **用途** | 调试阶段 | 发布阶段 |
| **优化** | 不优化（便于调试） | 优化速度和代码大小 |
| **调试信息** | 包含（.pdb 记录断点等信息） | 不包含（可单独生成 .pdb） |
| **生成文件** | .exe/.dll + .pdb | .exe/.dll |

### 2. 运行库选择（/MT、/MTd、/MD、/MDd）

| 选项 | 含义 | 链接方式 | 特点 |
|------|------|----------|------|
| **/MT** | MultiThread | 静态（LIBCMT.lib） | 程序可独立运行 |
| **/MTd** | MultiThread + Debug | 静态（LIBCMTD.lib） | Debug 版静态链接 |
| **/MD** | MultiThread + DLL | 动态（MSVCRT.dll） | 需目标机器有对应 DLL |
| **/MDd** | MultiThread + DLL + Debug | 动态（MSVCRTD.dll） | Debug 版动态链接 |

> /MT 编译的程序可直接移植到其他机器；/MD 编译的程序需要目标机器有对应的运行时 DLL。

## 四、VS 项目类型

Windows 提供 Win32 API 供程序完成各种操作。控制台、Win32、MFC 项目都可调用 Win32 API，区别在于界面和封装程度。

| 项目类型 | 界面 | 程序入口 | 说明 |
|----------|------|----------|------|
| **控制台应用** | 命令行（黑窗口） | `main` | subsystem = `console` |
| **空项目** | 无 | 自行定义 | 不含任何源文件，需自行添加 |
| **Win32 桌面应用** | 有窗口，控件需自己实现 | `WinMain` | subsystem = `windows`，直接调用 Win32 API |
| **MFC 应用** | 有窗口，控件可拖拽使用 | MFC 框架提供 | Win32 工程 + MFC 类库封装 |
