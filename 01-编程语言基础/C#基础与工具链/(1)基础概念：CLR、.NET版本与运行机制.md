# CLR、.NET 版本与运行机制

## 一、C# 语言概述

C#（读作 "C Sharp"）是微软于 2002 年发布的面向对象编程语言，运行在 .NET 平台上，由 Anders Hejlsberg 主导设计。

### C# 的特点

| 特点 | 说明 |
|------|------|
| **面向对象** | 封装、继承、多态 |
| **强类型** | 编译时类型检查，支持类型推断（var） |
| **自动内存管理** | 内置垃圾回收（GC） |
| **跨平台** | .NET 6+ 支持 Windows、Linux、macOS |
| **现代语法** | async/await、LINQ、模式匹配 |
| **丰富的生态** | NuGet 包管理、官方框架支持 |

### 应用领域

- 桌面应用（WPF、WinForms、MAUI）
- Web 应用（ASP.NET Core）
- 游戏开发（Unity）
- 云原生与微服务
- 移动应用（MAUI、Xamarin）
- IoT 与嵌入式

## 二、编译型 vs 解释型：C# 的混合模式

C# 与 Java 类似，采用**先编译后解释**的混合模式：

```
C# 源码(.cs)
    ↓  Roslyn（编译器）
中间语言 IL(.dll/.exe)
    ↓  CLR（公共语言运行时）
机器码执行（JIT 即时编译）
```

### 关键区别

| 特性 | C/C++ | Python | Java | C# |
|------|-------|--------|------|-----|
| **编译方式** | 直接编译为机器码 | 解释执行 | 编译为字节码 | 编译为 IL |
| **执行方式** | 直接运行 | 解释器逐行执行 | JVM 解释 + JIT | CLR 解释 + JIT |
| **跨平台** | 需重新编译 | 解释器跨平台 | 字节码跨平台 | IL 跨平台 |
| **运行速度** | 最快 | 较慢 | 接近 C/C++ | 接近 C/C++ |

## 三、CLR（公共语言运行时）

### 什么是 CLR？

CLR（Common Language Runtime）是 .NET 的运行时环境，负责执行 IL 代码，类似于 Java 的 JVM。

### CLR 的组成

```
┌─────────────────────────────────┐
│           CLR 架构              │
├─────────────────────────────────┤
│  程序集加载器（Assembly Loader）  │
│  → 加载 .dll / .exe 文件         │
├─────────────────────────────────┤
│  运行时数据区                     │
│  ├── 托管堆（Managed Heap）       │
│  ├── 线程栈（Thread Stack）       │
│  └── GC 堆（Generation 0/1/2）   │
├─────────────────────────────────┤
│  执行引擎                        │
│  ├── JIT 编译器：IL → 机器码      │
│  ├── 垃圾回收器：自动内存管理      │
│  └── 异常处理：结构化异常处理      │
└─────────────────────────────────┘
```

### CLR 的核心功能

| 功能 | 说明 |
|------|------|
| **JIT 编译** | 将 IL 编译为本机机器码 |
| **垃圾回收（GC）** | 自动管理内存，分代回收 |
| **类型安全** | 运行时类型检查 |
| **异常处理** | 结构化异常处理（try/catch/finally） |
| **线程管理** | 线程池、任务并行库（TPL） |

## 四、.NET 框架体系

### .NET 的命名演变

| 名称 | 年代 | 说明 |
|------|------|------|
| **.NET Framework** | 2002-2019 | 初代版本，仅 Windows，已停止更新 |
| **.NET Core** | 2016-2020 | 跨平台重写版，开源 |
| **.NET 5/6/7/8/9** | 2020-至今 | 统一后直接叫 ".NET" |

> **现在学什么？** 直接学 **.NET 8**（LTS）或 **.NET 9**（最新）。不要学 .NET Framework。

### .NET 框架对比

| 框架 | 平台 | 开源 | 状态 |
|------|------|------|------|
| **.NET Framework** | 仅 Windows | 否 | 维护模式 |
| **.NET Core** | 跨平台 | 是 | 已合并为 .NET 5+ |
| **.NET 5+** | 跨平台 | 是 | 当前主流 |
| **Mono** | 跨平台 | 是 | 用于 Xamarin/Unity |

## 五、JDK、JRE 与 JVM 的关系 → .NET 的对应

如果你学过 Java，可以这样理解：

| Java 概念 | .NET 对应 | 说明 |
|-----------|-----------|------|
| **JVM** | CLR | 运行时，执行中间代码 |
| **字节码（.class）** | IL（.dll/.exe） | 中间语言 |
| **JDK** | .NET SDK | 开发工具包 |
| **JRE** | .NET Runtime | 运行时（SDK 已包含） |
| **Maven/Gradle** | NuGet + MSBuild | 包管理 + 构建工具 |
| **pom.xml** | .csproj | 项目配置文件 |

```
.NET SDK（开发工具包）
├── CLR（运行时）
├── BCL（基础类库）
├── Roslyn（编译器）
├── dotnet CLI（命令行工具）
└── NuGet（包管理器）
```

## 六、版本演进

### .NET 版本历史

| 版本 | 发布年份 | 重要特性 |
|------|----------|----------|
| **.NET Framework 1.0** | 2002 | 首个版本 |

## 七、C# 的执行方式

### 1. 命令行编译运行

```bash
# 创建项目
dotnet new console -n HelloApp

# 运行
dotnet run
```

### 2. 交互式工具（dotnet-script）

```bash
# 安装
dotnet tool install -g dotnet-script

# 启动 REPL
dotnet-script

> Console.WriteLine("Hello!");
Hello!
```
