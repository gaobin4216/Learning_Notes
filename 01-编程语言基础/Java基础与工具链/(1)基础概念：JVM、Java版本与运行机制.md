# JVM、Java 版本与运行机制

## 一、Java 语言概述

Java 是由 Sun Microsystems（后被 Oracle 收购）于 1995 年发布的面向对象编程语言，核心理念是 **"Write Once, Run Anywhere"（一次编写，到处运行）**。

### Java 的特点

| 特点 | 说明 |
|------|------|
| **跨平台** | 通过 JVM 实现平台无关性 |
| **面向对象** | 封装、继承、多态 |
| **强类型** | 编译时类型检查 |
| **自动内存管理** | 内置垃圾回收（GC） |
| **多线程** | 原生支持并发编程 |
| **丰富的生态** | 海量开源框架和库 |

### 应用领域

- 企业级应用（Spring Boot、Jakarta EE）
- Android 应用开发
- 大数据处理（Hadoop、Spark、Flink）
- 分布式系统与微服务
- 金融交易系统
- Web 后端服务

## 二、编译型 vs 解释型：Java 的混合模式

Java 既不是纯粹的编译型语言，也不是纯粹的解释型语言，而是**先编译后解释**的混合模式：

```
Java 源码(.java)
    ↓  javac（编译器）
字节码(.class)
    ↓  JVM（Java 虚拟机）
机器码执行（JIT 即时编译）
```

### 关键区别

| 特性 | C/C++ | Python | Java |
|------|-------|--------|------|
| **编译方式** | 直接编译为机器码 | 解释执行 | 编译为字节码 |
| **执行方式** | 直接运行 | 解释器逐行执行 | JVM 解释 + JIT 编译 |
| **跨平台** | 需重新编译 | 解释器跨平台 | 字节码跨平台 |
| **运行速度** | 最快 | 较慢 | 接近 C/C++ |
| **源码保护** | 编译后难反编译 | 源码可直接查看 | 字节码可反编译 |

## 三、JVM（Java 虚拟机）

### 什么是 JVM？

JVM（Java Virtual Machine）是一个虚拟的计算机，负责执行 Java 字节码。它是 Java 跨平台能力的核心。

### JVM 的组成

```
┌─────────────────────────────────┐
│           JVM 架构              │
├─────────────────────────────────┤
│  类加载器（ClassLoader）         │
│  → 加载 .class 文件              │
├─────────────────────────────────┤
│  运行时数据区                     │
│  ├── 堆（Heap）：对象实例         │
│  ├── 方法区：类信息、常量池        │
│  ├── 虚拟机栈：方法调用栈帧        │
│  ├── 本地方法栈：Native 方法       │
│  └── 程序计数器：当前执行位置       │
├─────────────────────────────────┤
│  执行引擎                        │
│  ├── 解释器：逐条解释字节码        │
│  ├── JIT 编译器：热点代码编译      │
│  └── 垃圾回收器：自动内存管理      │
└─────────────────────────────────┘
```

### 常见 JVM 实现

| JVM | 说明 |
|-----|------|
| **HotSpot** | Oracle/OpenJDK 默认，最广泛使用 |
| **OpenJ9** | Eclipse 基金会，IBM 赞助 |
| **GraalVM** | Oracle 开发，支持多语言 |
| **Azul Zing** | 商业版，低延迟 GC |

## 四、JDK、JRE 与 JVM 的关系

```
JDK（Java Development Kit）
├── JRE（Java Runtime Environment）
│   ├── JVM（Java Virtual Machine）
│   ├── 核心类库（rt.jar / modules）
│   └── 其他运行时支持
├── javac（编译器）
├── jar（打包工具）
├── javadoc（文档生成）
├── jdb（调试器）
└── 其他开发工具
```

| 组件 | 包含内容 | 适合人群 |
|------|----------|----------|
| **JDK** | JRE + 开发工具 | 开发者 |
| **JRE** | JVM + 核心类库 | 只需运行 Java 程序的用户 |
| **JVM** | 字节码执行引擎 | （JRE 的一部分） |

> **注意**：从 JDK 9 开始，JDK 采用模块化系统（JPMS），不再单独提供 JRE 下载。从 JDK 11 开始，Oracle 不再提供免费的公共 JRE。

## 五、Java 版本演进

### 版本历史

| 版本 | 发布年份 | 重要特性 |
|------|----------|----------|
| **1.0** | 1996 | 首个正式版本 |
| **1.2** | 1998 | 集合框架、Swing |
| **1.4** | 2002 | 正则表达式、NIO |
| **5** | 2004 | 泛型、注解、自动装箱、枚举 |
| **6** | 2006 | 性能优化、脚本引擎 |
| **7** | 2011 | try-with-resources、钻石运算符 |
| **8（LTS）** | 2014 | Lambda、Stream API、Optional |
| **9** | 2017 | 模块化系统（JPMS） |
| **11（LTS）** | 2018 | var 局部变量、HTTP Client |
| **17（LTS）** | 2021 | 密封类、模式匹配、record |
| **21（LTS）** | 2023 | 虚拟线程、字符串模板、分代 ZGC |

> **LTS（Long-Term Support）** 版本提供长期支持，生产环境推荐使用 LTS 版本。

### 版本发布节奏

从 Java 9 开始，每 6 个月发布一个新版本（3 月和 9 月），每 3 年发布一个 LTS 版本。

## 六、Java 的执行方式

### 1. 命令行编译运行

```bash
# 编译
$ javac HelloWorld.java

# 运行
$ java HelloWorld
```

### 2. 交互式工具（JShell，Java 9+）

```bash
$ jshell
|  Welcome to JShell -- Version 21
|  Type for help

jshell> int x = 10
x ==> 10

jshell> x * 2
$2 ==> 20

jshell> /exit
```

### 3. 单文件源码运行（Java 11+）

```bash
# 无需先编译，直接运行
$ java HelloWorld.java
```
