# 工具箱、包管理与外部库集成

## 一、工具箱（Toolbox）

### 什么是工具箱？

工具箱是 MathWorks 官方发布的**专业领域扩展包**，每个工具箱包含一组相关函数、App 和文档。MATLAB 本身只提供基础计算能力，专业功能靠工具箱扩展。

### 工具箱的安装方式

工具箱在安装 MATLAB 时选择，也可以后续通过附加功能管理器安装：

```
MATLAB 主页 → 附加功能 → 获取附加功能
```

## 二、File Exchange（社区代码）

### 什么是 File Exchange？

[MathWorks File Exchange](https://www.mathworks.com/matlabcentral/fileexchange/) 是 MathWorks 官方的**社区代码分享平台**，用户可以免费下载各类 MATLAB 函数、工具和 App。

### 与工具箱的区别

| 对比 | 工具箱 | File Exchange |
|------|--------|---------------|
| **来源** | MathWorks 官方 | 社区用户 |
| **质量保证** | 官方测试和维护 | 质量参差不齐 |
| **价格** | 付费 | 免费 |
| **文档** | 完善 | 取决于作者 |
| **更新** | 随 MATLAB 版本更新 | 取决于作者 |

### 使用流程

1. 在 File Exchange 网站搜索需要的功能
2. 下载代码（ZIP 格式）
3. 解压到项目目录
4. 添加到 MATLAB 搜索路径

```matlab
addpath('下载的文件夹路径');       % 临时添加
addpath(genpath('my_project'));   % 递归添加所有子目录
savepath;                         % 永久保存路径
```

## 三、路径管理

### 为什么路径管理很重要？

MATLAB 通过**搜索路径**查找函数文件。如果你写的函数文件不在搜索路径中，MATLAB 就找不到它。

### 路径管理命令

| 命令 | 作用 |
|------|------|
| `path` | 查看当前搜索路径 |
| `addpath('dir')` | 添加目录到搜索路径 |
| `addpath(genpath('dir'))` | 递归添加目录及其所有子目录 |
| `rmpath('dir')` | 从搜索路径中移除目录 |
| `savepath` | 永久保存路径修改（写入 pathdef.m） |
| `restoredefaultpath` | 恢复到 MATLAB 默认搜索路径 |
| `which funcname` | 显示函数文件的实际路径 |
| `what` | 列出当前目录下的 MATLAB 文件 |

### `genpath` 的作用

`addpath` 只添加指定目录，不包含子目录。`genpath` 会生成包含所有子目录的路径字符串：

```matlab
% 只添加 src 目录，不包含子目录
addpath('src');

% 添加 src 目录及其所有子目录
addpath(genpath('src'));
```

## 四、外部语言互操作

### Python 互操作

```matlab
% 在 MATLAB 中调用 Python
result = py.math.sqrt(16);                    % 调用 Python 内置函数
np = py.importlib.import_module('numpy');     % 导入 NumPy
arr = np.array({1, 2, 3});                    % 创建 NumPy 数组

% 指定 Python 版本
pyenv('Version', 'C:/Python312/python.exe');

% 查看 Python 环境信息
pyenv
```

### Java 互操作

MATLAB 内置 JVM，可以直接使用 Java 类库：

```matlab
% 使用 Java 类
list = java.util.ArrayList();
list.add('hello');
list.add('world');

% 添加第三方 JAR 包
javaaddpath('第三方库.jar');

% 查看已加载的 Java 类
javaclasspath
```

### C/C++ MEX 文件

MEX 文件是用 C/C++ 编写的函数，编译后可以在 MATLAB 中直接调用，适用于性能关键的代码：

```matlab
% 编译 MEX 文件
mex myadd.c

% 调用（和普通 MATLAB 函数一样）
result = myadd(3, 4);
```

> **什么时候用 MEX？** 当 MATLAB 代码中有无法向量化的循环，且性能是瓶颈时，可以用 C/C++ 重写并编译为 MEX 文件，通常能获得 10-100 倍的加速。

## 五、项目依赖管理建议

MATLAB 没有像 pip、NuGet 那样的包管理器，依赖管理需要手动处理。推荐的项目结构：

```
myproject/
├── src/              % 源代码
├── lib/              % 第三方库（File Exchange 下载的）
│   ├── toolbox1/
│   └── toolbox2/
├── data/             % 数据文件
│   ├── raw/          % 原始数据（不修改）
│   └── processed/    % 处理后数据
├── results/          % 输出结果
├── tests/            % 测试脚本
├── setup_paths.m     % 路径初始化脚本
└── README.md
```

### 路径初始化脚本

```matlab
% setup_paths.m — 团队成员 clone 后运行一次即可
function setup_paths()
    root = fileparts(mfilename('fullpath'));
    addpath(genpath(fullfile(root, 'src')));
    addpath(genpath(fullfile(root, 'lib')));
end
```

> **团队协作**：每个成员 clone 项目后，运行一次 `setup_paths`，就能把所有需要的路径添加到 MATLAB 搜索路径中。
