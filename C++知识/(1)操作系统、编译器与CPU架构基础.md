- **BSD**：伯克利软件套件（Berkeley Software Distribution），由加州大学伯克利分校开发。BSD 拥护软件开源，鼓励代码共享。在 BSD 版本上开发商业版也是允许的，比如苹果的操作系统基于 BSD UNIX，但苹果的系统是闭源的。

### 5. GNU 与 GNU/Linux

#### GNU（GNU's Not Unix）

GNU 计划由理查德·斯托曼（Richard Stallman）在 1983 年发起，旨在开发一套完全自由的操作系统。GNU 是"GNU's Not Unix"的递归缩写，由一系列应用程序、系统库和开发工具构成的软件集合，再加上用于资源分配和硬件管理的内核。

为保证 GNU 软件可以自由地"使用、复制、修改和发布"，所有 GNU 软件都包含 GNU 通用公共许可证（GNU General Public License，GPL），即"公共版权"的概念。

> GNU 自己的内核 Hurd 仍在开发中，离实用还有一定距离。

#### GNU/Linux

1991 年 Linus Torvalds 编写出与 Unix 兼容的 Linux 操作系统内核并在 GPL 条款下发布。1992 年 Linux 与其他 GNU 软件结合，完全自由的操作系统正式诞生。

因此，现在的"Linux 系统"严格来说是 **GNU/Linux**——GNU 工具 + Linux 内核。许多 Unix 系统上也安装了 GNU 软件，GNU 工具还被广泛移植到 Windows 和 macOS 上。

## 二、编译器

### 1. 什么是编译器

编译器负责将高级语言（C、C++、Go 等）翻译成低级语言。现代编译器的主要工作流程：

```
源代码 → 预处理器 → 编译器 → 目标代码 → 链接器 → 可执行程序
```

> **编译型 vs 解释型**：C/C++ 需要编译成机器语言再链接成可执行文件；Python 则由解释器将源代码转为字节码后直接执行，无需手动编译和链接。

**编译器的核心构成**：

```
前端 → AST（抽象语法树）→ 后端
```

- **前端**：解析源码，生成 AST
- **后端**：将 AST 转为目标平台的机器码

### 2. 常见 C/C++ 编译器

| 编译器 | 说明 |
|--------|------|
| **MSVC** | 微软 Visual Studio 自带的 C/C++ 编译器，对 Windows 平台支持好，编译快 |
| **GCC** | GNU Compiler Collection，GNU 开发的编译器集，支持 C/C++/Ada/Objective-C 等多种语言，支持 X86/ARM/MIPS 等多种架构，类 Unix 下的标准编译器 |
| **LLVM/Clang** | LLVM 是开源编译器架构，Clang 是其 C/C++/Objective-C 前端。由苹果赞助开发，设计优良，适合源码级分析工具开发 |

## 三、CPU 架构

CPU（中央处理单元）主要由运算器、控制器、寄存器组成，负责处理指令、执行操作、控制时间、处理数据。CPU 架构是区分不同类型 CPU 的重要标识。

目前市面上的 CPU 分两大阵营：
- **复杂指令集（CISC）**：Intel、AMD 为首
- **精简指令集（RISC）**：IBM、ARM 为首

### 1. 指令集（逻辑分类）

#### CISC（复杂指令集）

每个指令可执行若干低阶操作（读写内存、计算等），全部集于单一指令之中。

特点：
1. 指令系统庞大，指令功能复杂，指令格式、寻址方式多
2. 绝大多数指令需多个机器周期完成
3. 各种指令都可访问存储器
4. 采用微程序控制
5. 专用寄存器，数量少
6. 难以用优化编译技术生成高效目标代码

> 常见 CISC：AMD、Intel、VIA 等 IA-32/X86 架构 CPU。
> 约 20% 的指令被反复使用，占程序代码的 80%；其余 80% 的指令只占程序的 20%。

#### RISC（精简指令集）

对指令数目和寻址方式做了精简，使实现更容易，指令并行执行程度更好，编译器效率更高。

特点：
1. 统一指令编码（op-code 位置固定、等长指令），可快速解译
2. 泛用寄存器，所有寄存器可用于所有内容
3. 单纯的寻址模式（复杂寻址以简单计算指令序列取代）
4. 硬件中仅支持少数数据类型

> 常见 RISC：DEC Alpha、ARC、ARM、AVR、MIPS、PA-RISC、IBM Power（含 PowerPC）、SUN SPARC。

### 2. 常见架构

#### X86 架构
> 应用领域：个人计算机、服务器等

x86 是英特尔首先开发制造的微处理器体系结构泛称，因早期处理器以"86"结尾（8086、80286、80386 等）而得名。

**关键演进**：
- **x86 / IA-32**：1978 年 Intel 8086 首度出现，1985 年 80386 开始支持 32 位
- **IA-64**：Intel 与惠普合作的全新 64 位架构，与 x86 不兼容，市场惨淡
- **x86-64 / X64**：2003 年 AMD 推出的兼容 32 位的 64 位扩展（AMD64），Intel 后来也采用，称为 Intel 64

> 查询 CPU 位数：出现 x86/i386/i586 为 32 位；出现 AMD64/x64 为 64 位。

**64 位的优势**：
- 通用寄存器位宽翻倍，可一次性处理 64bit 整型数据
- 内存寻址能力更强：32 位 CPU 理论上限约 4GB，64 位理论上可达 2^64 字节

#### ARM 架构
> 应用领域：手机、平板、导航系统、网络设备等，目前世界上使用最广泛的 CPU 架构

ARM（Advanced RISC Machine）是 32 位精简指令集处理器架构，1991 年成立于英国剑桥，主要出售芯片设计技术授权。由于节能特点，ARM 非常适用于移动通讯领域，占所有 32 位嵌入式处理器 75% 的比例。

#### MIPS 架构
> 应用领域：工作站等计算机平台，目前国内龙芯主要使用 MIPS 架构

MIPS（Microprocessor without Interlocked Pipeline Stages）是 1981 年出现的 RISC 处理器架构，由 MIPS 科技公司开发并授权。强调软硬件协同提高性能，同时简化硬件设计。

> 2002 年中国科学院计算所开始研发龙芯处理器，采用 MIPS 架构，2007 年获得正式授权。

#### PowerPC 架构

PowerPC 是 IBM、Apple、Motorola 三家公司于 1990 年代开发的 RISC 架构 CPU。特点是可伸缩性好、方便灵活，实现范围从高端服务器 CPU 到嵌入式 CPU（如任天堂 GameCube）。
