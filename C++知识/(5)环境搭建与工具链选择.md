# 环境搭建与工具链选择

## 一、包管理器对比

| 包管理器 | 平台 | 管理范围 | 说明 |
|----------|------|----------|------|
| **pacman** | MSYS2 / Arch Linux | 系统软件包（编译工具链、开发工具、库等） | MSYS2 的核心优势，一行命令装库 |
| **apt** | Debian / Ubuntu | 系统软件包 | Linux 下最常用的包管理器 |
| **vcpkg** | 跨平台 | 专注 C++ 第三方库 | 微软开发，与 CMake 集成 |

> **核心区别**：pacman/apt 管理整个系统（含编译器、工具、库），vcpkg 只管 C++ 第三方库。

---

## 二、工具链搭配

> 不同编译器编译的库存在ABI差异，vcpkg 用 MSVC 编译的库，用 GCC 调用会出问题。**不要混搭**。

### Windows 上的两种主流搭配

| 搭配方案 | 编译器 | 包管理 | 构建工具 | 适合场景 |
|----------|--------|--------|----------|----------|
| **VS Code + MSYS2** | GCC（MinGW-w64） | pacman | CMake + mingw32-make | 轻量开发，类 Linux 体验 |
| **Visual Studio + vcpkg** | MSVC | vcpkg | Windows 版 CMake | 重量级项目，VS 全家桶 |

### VS Code + MSYS2（推荐新手）

1. 安装 MSYS2，用 pacman 安装 MinGW-w64 工具链
2. 环境变量设为 `C:/msys64/mingw64/bin`
3. VS Code 中选择 MSYS2 下的 gcc 作为编译器
4. 用 CMake 构建，`make` 命令实际是 `mingw32-make`
5. 第三方库用 pacman 安装：`pacman -S mingw-w64-ucrt-x86_64-<包名>`

### Visual Studio + vcpkg

1. 安装 Visual Studio（至少 Build Tools，勾选"C++桌面开发"和"英语"）
2. 安装 vcpkg，用 Windows 版 CMake 编译
3. VS Code 中编译器选 MSVC（vcpkg 编译的库适用于 MSVC）
4. vcpkg 与 VS 无缝集成，自动配置头文件和库路径

> **总结**：vcpkg 适合搭配 VS + MSVC；VS Code + GCC 用 pacman 管理库更省心。

## 三、Windows 环境搭建

### 使用 MSYS2

- 安装 MSYS2，用 pacman 安装 MinGW 和 GCC，安装第三方库
- 环境变量设为 `C:/msys64/mingw64/bin`
- VS Code 中配置工具为 MSYS2 下的 gcc
- 使用 CMake 构建，`make` 命令是 `mingw32-make`

### 第三方库管理对比

| 方式 | 流程 | 体验 |
|------|------|------|
| **传统方式** | 去官网下载 → 解压 → 手动配置 include/lib → 版本不兼容 → 报错 | 痛苦 |
| **MSYS2 pacman** | `pacman -S mingw-w64-ucrt-x86_64-opencv`，CMake 自动找到 | 零配置 |
| **Linux apt** | `apt install libopencv-dev`，CMake 自动识别 | 同样零配置 |

## 四、Linux 环境搭建

标准项目结构：

```
project/
├── CMakeLists.txt
├── include/
├── src/
└── build/
```

构建命令：

```bash
cd build
cmake ..
make
./可执行文件名
```

> Linux 下 `apt install` 装完开发库后，CMake 自动识别路径，无需手动配置。体验和 MSYS2 一致。
