# 环境搭建与工具链选择

## 一、包管理器对比

| 包管理器 | 平台 | 管理范围 | 说明 |
|----------|------|----------|------|
| **pacman** | MSYS2 / Arch Linux | 系统软件包（编译工具链、开发工具、库等） | MSYS2 的核心优势，一行命令装库 |
| **apt** | Debian / Ubuntu | 系统软件包 | Linux 下最常用的包管理器 |
| **vcpkg** | 跨平台 | 专注 C++ 第三方库 | 微软开发，与 CMake 集成 |

> **核心区别**：pacman/apt 管理整个系统（含编译器、工具、库），vcpkg 只管 C++ 第三方库。

---

## 二、工具链搭配

> 不同编译器编译的库存在 ABI 差异，vcpkg 用 MSVC 编译的库用 GCC 调用会出问题。**不要混搭**。

| 搭配方案 | 编译器 | 包管理 | 构建工具 | 适合场景 |
|----------|--------|--------|----------|----------|
| **VS Code + MSYS2** | GCC（MinGW-w64） | pacman | CMake + mingw32-make | 轻量开发，类 Linux 体验（推荐新手） |
| **Visual Studio + vcpkg** | MSVC | vcpkg | Windows 版 CMake | 重量级项目，VS 全家桶 |
| **VS Code + WSL2** | GCC（Linux 原生） | apt | CMake + make | Linux 程序开发 |

---

## 三、Windows 环境搭建

### 方案一：VS Code + MSYS2

1. 安装 MSYS2，打开 UCRT64 终端
2. 安装工具链：`pacman -S mingw-w64-ucrt-x86_64-toolchain`
3. 安装 CMake：`pacman -S mingw-w64-ucrt-x86_64-cmake`
4. 将 `C:/msys64/ucrt64/bin` 加入系统环境变量 `PATH`
5. VS Code 安装 C/C++ 和 CMake Tools 插件，编译器选 UCRT64 的 gcc
6. 第三方库用 pacman 安装：`pacman -S mingw-w64-ucrt-x86_64-<包名>`

### 方案二：Visual Studio + vcpkg

1. 安装 Visual Studio（至少 Build Tools，勾选"C++桌面开发"和"英语"）
2. 克隆 vcpkg 并引导：`git clone https://github.com/microsoft/vcpkg && .\vcpkg\bootstrap-vcpkg.bat`
3. 安装库：`vcpkg install <包名>:x64-windows`
4. VS 中通过 `vcpkg integrate install` 自动配置头文件和库路径

### 第三方库管理对比

| 方式 | 流程 | 体验 |
|------|------|------|
| **传统方式** | 去官网下载 → 解压 → 手动配置 include/lib → 版本不兼容 | 痛苦 |
| **MSYS2 pacman** | `pacman -S mingw-w64-ucrt-x86_64-opencv`，CMake 自动找到 | 零配置 |
| **vcpkg** | `vcpkg install opencv`，VS 自动集成 | 自动配置 |

## 四、Linux 环境搭建

### 安装开发工具

```bash
sudo apt update
sudo apt install build-essential cmake gdb
# build-essential 包含 gcc、g++、make 等
```

### 标准项目结构

```
project/
├── CMakeLists.txt
├── include/        # 头文件
├── src/            # 源文件
└── build/          # 构建输出（不污染源码目录）
```

### 构建流程

```bash
mkdir build && cd build
cmake ..
make
./可执行文件名
```

### 第三方库

```bash
# 安装（以 OpenCV 为例）
sudo apt install libopencv-dev

# CMakeLists.txt 中直接使用
find_package(OpenCV REQUIRED)
target_link_libraries(MyApp ${OpenCV_LIBS})
```

> Linux 下 apt 装完开发库后，CMake 自动识别路径，无需手动配置。体验和 MSYS2 pacman 一致。
