## QT是啥


Qt（发音同“cute”）是一个功能强大的跨平台 C++ 应用程序开发框架。它于1991年诞生，最初主要用于开发图形用户界面（GUI），现在已经发展成一个能够构建桌面、移动、嵌入式及物联网应用的完整平台。简单来说，Qt 就像是一个“万能工具箱”，它为你提供了构建复杂应用所需的一切工具，让你能用一套代码，轻松部署到 Windows、macOS、Linux、Android、iOS 等多个平台上。

### 跨平台

*一套东西，换个平台打开cmakelist编译一下就行*

### 特点
* 真正的跨平台，这是 Qt 最核心的优势。你只需编写一次代码，就可以在不同的操作系统上编译运行，极大地提高了开发效率。
* 信号与槽（Signals & Slots），这是 Qt 独创且最著名的通信机制。它让对象之间的通信变得非常安全和简洁。信号（Signal）：当一个对象的状态发生变化时（比如按钮被点击），它会发出一个信号。槽（Slot）：另一个对象可以“监听”这个信号，并在接收到信号时执行相应的函数。这种机制有效地解耦了组件，让代码更清晰、更易维护。
* 丰富的功能模块，Qt 不仅仅是一个 GUI 库，它提供了极其丰富的模块，涵盖了应用开发的方方面面：
* GUI 模块：提供按钮、文本框、表格等大量标准控件。
* 网络模块：支持 HTTP、TCP、WebSocket 等网络协议。
* 数据库模块：提供统一的接口访问 SQLite、MySQL 等多种数据库。多线程、XML、OpenGL 等等，应有尽有。
* 现代化的 UI 技术 (QML)，除了传统的 C++ Widgets，Qt 还提供了 QML 和 Qt Quick 框架。QML 是一种声明式语言，特别适合开发具有流畅动画和现代感的用户界面，在移动端和嵌入式设备上表现尤为出色。
* 强大的开发工具 (Qt Creator)
Qt 官方提供了一个名为 Qt Creator 的集成开发环境（IDE）。它轻量、快速且功能强大，集成了代码编辑、UI 可视化设计、调试和项目管理等所有功能，极大地提升了开发体验。

### 版本
Qt 6 相比 Qt 5 是一次底层架构的重大升级，而不仅仅是简单的功能增加。虽然它们看起来很像，但在核心机制上有很多不同。

| 特性 | Qt 5 | Qt 6 | 影响 |
| :--- | :--- | :--- | :--- |
| C++ 标准 | C++11 / C++14 | C++17 (最低要求) | Qt 6 代码更简洁，利用了现代 C++ 特性（如结构化绑定）。 |
| 构建工具 | qmake (主导) | CMake (主导) | Qt 6 彻底拥抱 CMake，`qmake` 虽保留但已不再推荐用于新项目。 |
| 图形渲染 | 依赖 OpenGL | RHI (渲染硬件接口) | Qt 6 自动适配 Vulkan (PC/Android), Metal (Apple), Direct3D (Windows)，不再死磕 OpenGL。 |
| 3D 支持 | 独立的 Qt 3D 模块 | Qt Quick 3D | Qt 6 将 2D UI 和 3D 内容融合在同一个场景图中，混合开发更简单。 |

### qmake与cmake的区别
| 特性 | qmake | CMake |
| :--- | :--- | :--- |
| 定位 | Qt 专用构建工具 | 通用跨平台构建系统 (C/C++ 标准) |
| 配置文件 | `.pro` (简单，声明式) | `CMakeLists.txt` (强大，命令式/脚本) |
| 跨平台能力 | 较弱，主要生成 Makefile | 极强，生成 VS, Xcode, Ninja, Makefile 等 |
| 第三方库 | 需手动指定路径 (`INCLUDEPATH`, `LIBS`) | 自动查找 (`find_package`)，生态极好 |
| Qt 官方态度 | 维护模式 (仅用于旧项目) | 官方推荐 (Qt 6 默认且首选) |
| 上手难度 | ⭐ (非常简单) | ⭐⭐⭐ (语法较复杂，概念多) |
| 构建速度 | 一般 | 快 (配合 Ninja 后端速度极快) |

qmake：它是 Qt 团队早年为了简化 Qt 开发而写的工具。它非常懂 Qt（比如自动处理 moc, uic, rcc），但除了 Qt 它什么都不懂。如果你想链接一个 OpenCV 或 Boost 库，你得手动写一堆路径配置。
CMake：它是整个 C++ 界的工业标准。无论你是做游戏、AI、嵌入式还是 Qt 开发，大家都在用 CMake。它拥有庞大的生态系统，绝大多数第三方库（如 vcpkg, conan）都优先提供 CMake 支持。

## Qt6 中 Qt Quick 与 Widgets
Qt Widgets：传统桌面 UI，像 Windows 原生窗口、按钮、表格，适合工具软件、后台程序、工业软件。
Qt Quick：现代流畅 UI，像手机 APP、动画界面、触控界面，适合高颜值、流畅、跨端界面。
| 对比维度 | Qt Widgets | Qt Quick (QML) |
| :--- | :--- | :--- |
| 核心语言 | C++ (逻辑与 UI 紧密耦合) | QML (声明式语法) + JavaScript/C++ |
| 渲染机制 | CPU 渲染 (基于光栅化，依赖系统原生控件) | GPU 加速 (基于 OpenGL/Vulkan/Metal 场景图) |
| 界面风格 | 传统桌面风格 (Windows/macOS/Linux 原生外观) | 现代化、扁平化、高度可定制 (类似移动端风格) |
| 开发模式 | 命令式 (代码控制流程，布局相对繁琐) | 声明式 (描述 UI 状态，支持热重载，开发快) |
| 数据绑定 | 手动更新 (通常使用信号与槽机制) | 自动属性绑定 (Property Binding，类似 Vue/React) |
| 动画特效 | 较弱 (实现复杂动画需大量代码，性能开销大) | 极强 (原生支持平滑动画、粒子效果、3D 变换) |
| 触摸支持 | 一般 (需手动处理，主要针对鼠标键盘优化) | 完美 (原生支持多点触控、手势、惯性滚动) |
| 资源占用 | 较低 (内存占用小，启动速度快) | 较高 (需加载 QML 引擎，显存占用相对较高) |
| 适用平台 | 桌面端 (Windows, Linux, macOS) | 全平台 (桌面 + 移动端 iOS/Android + 嵌入式) |
