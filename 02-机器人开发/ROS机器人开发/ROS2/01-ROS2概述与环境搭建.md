# ROS2概述与环境搭建

---

# 一、ROS2是什么？

ROS2是ROS（Robot Operating System）的**第二代版本**，从2017年开始开发，2022年ROS1停止维护（Noetic EOL）。ROS2不是ROS1的简单升级，而是**重新设计**的架构，解决了ROS1的诸多局限性。

**ROS2整体架构：**
```
┌─────────────────────────────────────────────────────┐
│                   应用层                              │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌────────┐ │
│  │  MoveIt2 │ │Navigation│ │  SLAM2   │ │ 自定义  │ │
│  └────┬─────┘ └────┬─────┘ └────┬─────┘ └───┬────┘ │
├───────┼────────────┼────────────┼────────────┼───────┤
│       │         ROS2 中间件层       │            │       │
│  ┌────┴────────────┴────────────┴────────────┴────┐ │
│  │              rclcpp / rclpy (客户端库)          │ │
│  ├─────────────────────────────────────────────────┤ │
│  │                    RCL (核心库)                  │ │
│  ├─────────────────────────────────────────────────┤ │
│  │                    rmw (中间件接口)              │ │
│  ├─────────────────────────────────────────────────┤ │
│  │         DDS实现层 (Fast DDS / Cyclone DDS)       │ │
│  └─────────────────────────────────────────────────┘ │
├───────────────────────────────────────────────────────┤
│                   操作系统层                            │
│            Linux / Windows / macOS                     │
└───────────────────────────────────────────────────────┘
```

**ROS2 vs ROS1 的核心变化：**

| 对比项 | ROS1 | ROS2 |
|--------|------|------|
| 通信机制 | 自定义TCP/UDP，依赖Master | 基于DDS（数据分发服务），去中心化 |
| 实时性 | 不支持 | 支持实时控制 |
| 跨平台 | 仅Linux | Linux/Windows/macOS |
| 安全性 | 无 | 内置DDS安全机制 |
| 多机器人 | 困难 | 原生支持命名空间隔离 |
| 编译系统 | catkin | ament + colcon |

**为什么学习ROS2？**
- ROS1已于2023年停止更新，ROS2是未来
- Jazzy是当前推荐的LTS版本（长期支持到2029年）
- 工业界越来越多地采用ROS2

# 二、ROS2版本与发行策略

ROS2采用**每年两个版本**的发布节奏，分为普通版和LTS（长期支持）版：

| 版本代号 | 发布时间 | Ubuntu版本 | 支持状态 |
|----------|----------|------------|----------|
| Humble | 2022.05 | 22.04 | LTS，支持到2027.05 |
| Iron | 2023.05 | 22.04 | EOL 2024.11 |
| **Jazzy** | **2024.05** | **24.04** | **LTS，支持到2029.05** |
| Kilted | 2025.05 | 24.04 | 最新版本 |

**建议**：新手选择 **Jazzy**（LTS），稳定性最好，社区支持最多。

# 三、安装ROS2 Jazzy

## 前置条件
- Ubuntu 24.04 LTS（Noble Numbat）
- 确保系统已更新：`sudo apt update && sudo apt upgrade`

## 安装步骤

```bash
# 1. 设置locale
sudo apt update && sudo apt install locales
sudo locale-gen en_US en_US.UTF-8
sudo update-locale LC_ALL=en_US.UTF-8 LANG=en_US.UTF-8
export LANG=en_US.UTF-8

# 2. 添加ROS2软件源
sudo apt install software-properties-common
sudo add-apt-repository universe

# 添加ROS2 GPG密钥
sudo apt update && sudo apt install curl -y
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key -o /usr/share/keyrings/ros-archive-keyring.gpg

# 添加软件源
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] http://packages.ros.org/ros2/ubuntu $(. /etc/os-release && echo $UBUNTU_CODENAME) main" | sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null

# 3. 安装ROS2
sudo apt update
sudo apt upgrade
sudo apt install ros-jazzy-desktop

# 4. 设置环境变量
echo "source /opt/ros/jazzy/setup.bash" >> ~/.bashrc
source ~/.bashrc

# 5. 安装开发工具（可选但推荐）
sudo apt install python3-colcon-common-extensions python3-rosdep
sudo rosdep init
rosdep update
```

## 验证安装

```bash
# 终端1：运行小海龟仿真（ROS2经典示例）
ros2 run turtlesim turtlesim_node

# 终端2：用键盘控制海龟
ros2 run turtlesim turtle_teleop_key
```

如果小海龟能动，说明ROS2安装成功！

# 四、ROS2与ROS1的主要区别（快速了解）

> 详细对比见 `02-ROS2与ROS1对比.md`

### 1、不再需要roscore
ROS2去掉了ROS Master，节点之间通过DDS自动发现。不需要先启动任何核心进程。

### 2、命令行工具变了
```bash
# ROS1
rosrun turtlesim turtlesim_node
rostopic list
rosnode list

# ROS2
ros2 run turtlesim turtlesim_node
ros2 topic list
ros2 node list
```

### 3、编译系统换了
```bash
# ROS1：catkin_make
cd ~/catkin_ws && catkin_make

# ROS2：colcon build
cd ~/ros2_ws && colcon build
```

### 4、Python不再需要CMakeLists.txt
ROS2的Python包只需要 `setup.py` 和 `package.xml`，更简洁。

# 五、创建工作空间

## ROS2工作空间结构

```
ros2_ws/
├── src/            # 源代码目录
│   ├── my_pkg/     # 你的功能包
│   └── ...
├── build/          # 编译中间文件（自动生成）
├── install/        # 安装文件（自动生成）
└── log/            # 编译日志（自动生成）
```

## 创建步骤

```bash
# 1. 创建工作空间目录
mkdir -p ~/ros2_ws/src
cd ~/ros2_ws

# 2. 编译（初始为空，会生成build/install/log目录）
colcon build

# 3. 设置环境变量
echo "source ~/ros2_ws/install/setup.bash" >> ~/.bashrc
source ~/.bashrc
```

# 六、创建功能包

## Python包（推荐新手）
```bash
cd ~/ros2_ws/src
ros2 pkg create --build-type ament_python my_robot --dependencies rclpy std_msgs
```

生成的结构：
```
my_robot/
├── my_robot/           # Python源码目录
│   ├── __init__.py
│   └── my_node.py      # 你的节点代码
├── package.xml         # 包描述文件
├── setup.py            # Python安装配置
├── setup.cfg           # 配置文件
└── test/               # 测试目录
```

## C++包
```bash
cd ~/ros2_ws/src
ros2 pkg create --build-type ament_cmake my_robot --dependencies rclcpp std_msgs
```

生成的结构：
```
my_robot/
├── include/my_robot/   # 头文件
├── src/                # C++源码
│   └── my_node.cpp
├── CMakeLists.txt      # 编译配置
├── package.xml         # 包描述文件
└── test/               # 测试目录
```

## 编译与运行

```bash
# 编译整个工作空间
cd ~/ros2_ws
colcon build

# 只编译某个包
colcon build --packages-select my_robot

# 编译后设置环境
source install/setup.bash

# 运行节点
ros2 run my_robot my_node
```

# 七、常用命令速查

```bash
# ==================== 节点 ====================
ros2 run <包名> <节点名>              # 运行节点
ros2 node list                        # 查看所有节点
ros2 node info <节点名>               # 查看节点信息

# ==================== 话题 ====================
ros2 topic list                       # 查看所有话题
ros2 topic echo <话题名>              # 打印话题数据
ros2 topic pub <话题名> <类型> <数据>  # 发布话题
ros2 topic hz <话题名>                # 查看发布频率
ros2 topic info <话题名>              # 查看话题信息

# ==================== 服务 ====================
ros2 service list                     # 查看所有服务
ros2 service call <服务名> <类型> <参数> # 调用服务

# ==================== 动作 ====================
ros2 action list                      # 查看所有动作
ros2 action info <动作名>             # 查看动作信息
ros2 action send_goal <动作名> <类型> <目标> # 发送动作目标

# ==================== 参数 ====================
ros2 param list                       # 查看所有参数
ros2 param get <节点名> <参数名>      # 获取参数
ros2 param set <节点名> <参数名> <值> # 设置参数
ros2 param dump <节点名>              # 导出参数到文件

# ==================== 包 ====================
ros2 pkg list                         # 查看所有包
ros2 pkg executables <包名>           # 查看包的可执行文件

# ==================== 编译 ====================
cd ~/ros2_ws && colcon build          # 编译工作空间
colcon build --packages-select <包名> # 编译指定包
source install/setup.bash             # 设置环境变量

# ==================== Launch ====================
ros2 launch <包名> <launch文件>       # 启动launch文件
```
