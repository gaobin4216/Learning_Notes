# 一、colcon编译工具

## colcon是什么？

colcon是ROS2的**编构工具**，类似于ROS1的catkin_make，但功能更强：
- 支持同时编译多个包
- 支持并行编译
- 支持多种构建系统（ament_cmake、ament_python、cmake）

## 常用命令

```bash
# 编译整个工作空间
cd ~/ros2_ws
colcon build

# 编译指定包
colcon build --packages-select my_pkg

# 编译并设置符号链接（Python包修改后不用重新编译）
colcon build --symlink-install

# 编译并显示详细输出
colcon build --event-handlers console_direct+

# 运行所有测试
colcon test

# 运行指定包的测试
colcon test --packages-select my_pkg

# 查看测试结果
colcon test-result --verbose

# 编译后设置环境
source install/setup.bash
```

## 工作空间覆盖（Overlay）

ROS2支持工作空间覆盖，后source的工作空间会覆盖前面的：
```bash
source /opt/ros/jazzy/setup.bash        # 底层工作空间
source ~/ros2_ws/install/setup.bash      # 覆盖层
```

# 二、launch文件

## 为什么需要launch文件？

一个机器人系统通常有多个节点需要同时运行。launch文件可以一次性启动多个节点，设置参数，配置命名空间等。

## Python Launch（推荐）

ROS2推荐使用Python格式的launch文件，功能最灵活：

```python
# my_launch.py
from launch import LaunchDescription
from launch_ros.actions import Node
from launch.actions import DeclareLaunchArgument
from launch.substitutions import LaunchConfiguration

def generate_launch_description():
    return LaunchDescription([
        # 声明参数
        DeclareLaunchArgument(
            'use_sim_time',
            default_value='false',
            description='Use simulation time'
        ),
        
        # 启动节点
        Node(
            package='turtlesim',
            executable='turtlesim_node',
            name='my_turtle',
            parameters=[{'use_sim_time': LaunchConfiguration('use_sim_time')}],
            output='screen'
        ),
        
        Node(
            package='turtlesim',
            executable='turtle_teleop_key',
            name='teleop',
            output='screen'
        ),
    ])
```

**运行launch文件：**
```bash
ros2 launch my_pkg my_launch.py

# 带参数运行
ros2 launch my_pkg my_launch.py use_sim_time:=true
```

## Launch常用操作

```python
from launch import LaunchDescription
from launch_ros.actions import Node, ComposableNodeContainer
from launch.actions import (
    IncludeLaunchDescription,
    ExecuteProcess,
    TimerAction,
    GroupAction
)
from launch.launch_description_sources import PythonLaunchDescriptionSource
from launch_ros.descriptions import ComposableNode

def generate_launch_description():
    return LaunchDescription([
        # 1. 基本节点启动
        Node(
            package='my_pkg',
            executable='my_node',
            name='node_name',
            namespace='robot1',
            parameters=['config/params.yaml'],  # 从YAML加载参数
            remappings=[('/old_topic', '/new_topic')],  # 话题重映射
            output='screen'
        ),
        
        # 2. 包含其他launch文件
        IncludeLaunchDescription(
            PythonLaunchDescriptionSource(
                [FindPackageShare('other_pkg'), '/launch/other.launch.py']
            ),
            launch_arguments={'arg1': 'value1'}.items()
        ),
        
        # 3. 延迟启动（等待2秒后启动）
        TimerAction(
            period=2.0,
            actions=[
                Node(package='my_pkg', executable='delayed_node', output='screen')
            ]
        ),
        
        # 4. 执行shell命令
        ExecuteProcess(
            cmd=['echo', 'Hello ROS2!'],
            output='screen'
        ),
    ])
```

## YAML Launch（简单场景）

```yaml
# my_launch.yaml
launch:
  - node:
      pkg: turtlesim
      exec: turtlesim_node
      name: my_turtle
      output: screen

  - node:
      pkg: turtlesim
      exec: turtle_teleop_key
      name: teleop
      output: screen
```

```bash
ros2 launch my_pkg my_launch.yaml
```

## 参数文件（YAML）

```yaml
# config/params.yaml
/my_node:
  ros__parameters:
    robot_name: "my_robot"
    max_speed: 1.0
    debug: true
```

在launch文件中加载：
```python
Node(
    package='my_pkg',
    executable='my_node',
    parameters=['config/params.yaml']
)
```

# 三、RViz2可视化

## RViz2是什么？

RViz2是ROS2的三维可视化工具，用于显示机器人模型、传感器数据、TF变换等。它是ROS1中RViz的ROS2版本。

## 启动RViz2

```bash
# 直接启动
rviz2

# 带配置文件启动
rviz2 -d ~/my_config.rviz
```

## 常用显示类型

| 显示类型 | 用途 |
|----------|------|
| RobotModel | 显示URDF机器人模型 |
| TF | 显示坐标系变换 |
| LaserScan | 显示激光雷达数据 |
| Image | 显示相机图像 |
| PointCloud2 | 显示3D点云 |
| Path | 显示路径规划结果 |
| Marker | 显示自定义标记 |
| Pose | 显示位姿 |

## 在launch文件中启动RViz2

```python
from launch_ros.actions import Node
from ament_index_python.packages import get_package_share_directory
import os

Node(
    package='rviz2',
    executable='rviz2',
    name='rviz2',
    arguments=['-d', os.path.join(
        get_package_share_directory('my_pkg'), 'config', 'my.rviz'
    )],
    output='screen'
)
```

# 五、tf2坐标变换

## tf2是什么？

tf2（Transform Framework 2）是ROS2中的坐标变换库，负责管理机器人系统中所有坐标系之间的关系。

## 坐标系示例

```
world (世界坐标系)
  └── robot_base (机器人基座)
        ├── arm_base (手臂基座)
        │     ├── link1 (连杆1)
        │     └── link2 (连杆2)
        └── camera_link (相机)
```

## 发布TF变换

```python
from geometry_msgs.msg import TransformStamped
import tf2_ros

class TfBroadcaster(Node):
    def __init__(self):
        super().__init__('tf_broadcaster')
        self.br = tf2_ros.TransformBroadcaster(self)
        self.timer = self.create_timer(0.1, self.broadcast_tf)

    def broadcast_tf(self):
        t = TransformStamped()
        t.header.stamp = self.get_clock().now().to_msg()
        t.header.frame_id = 'world'
        t.child_frame_id = 'robot_base'
        t.transform.translation.x = 1.0
        t.transform.translation.y = 2.0
        t.transform.rotation.w = 1.0
        self.br.sendTransform(t)
```

## 监听TF变换

```python
import tf2_ros

class TfListener(Node):
    def __init__(self):
        super().__init__('tf_listener')
        self.tf_buffer = tf2_ros.Buffer()
        self.tf_listener = tf2_ros.TransformListener(self.tf_buffer, self)
        self.timer = self.create_timer(1.0, self.lookup_tf)

    def lookup_tf(self):
        try:
            trans = self.tf_buffer.lookup_transform(
                'world', 'robot_base', rclpy.time.Time()
            )
            self.get_logger().info(
                f'Position: ({trans.transform.translation.x}, '
                f'{trans.transform.translation.y})'
            )
        except tf2_ros.TransformException as e:
            self.get_logger().warn(f'变换查找失败: {e}')
```

## TF操作命令

```bash
# 查看所有TF关系
ros2 run tf2_tools view_frames

# 查看两个坐标系之间的变换
ros2 run tf2_ros tf2_echo world robot_base
```

# 六、Gazebo仿真

## Gazebo版本说明

ROS2 Jazzy支持两个版本的Gazebo：
- **Gazebo Classic（Gazebo 11）**：旧版，但仍可用
- **Gazebo（Ignition → Gazebo）**：新版，推荐使用

## 启动Gazebo

```bash
# 安装Gazebo
sudo apt install ros-jazzy-gazebo-ros-pkgs

# 启动空世界
ros2 launch gazebo_ros gazebo.launch.py

# 启动带机器人的仿真
ros2 launch my_robot_description gazebo.launch.py
```

## 在Gazebo中加载URDF

```python
# launch文件示例
from launch import LaunchDescription
from launch_ros.actions import Node
from launch.actions import ExecuteProcess

def generate_launch_description():
    return LaunchDescription([
        # 启动Gazebo
        ExecuteProcess(
            cmd=['gazebo', '--verbose', '-s', 'libgazebo_ros_factory.so'],
            output='screen'
        ),
        
        # 在Gazebo中生成机器人
        Node(
            package='gazebo_ros',
            executable='spawn_entity.py',
            arguments=[
                '-topic', 'robot_description',
                '-entity', 'my_robot'
            ],
            output='screen'
        ),
    ])
```

# 七、ros2_control控制器框架

## ros2_control是什么？

ros2_control是ROS2中的硬件抽象和控制框架，是ROS1中ros_control的ROS2版本。它负责将上层规划（如MoveIt2）的结果发送给实际硬件或仿真器。

## 核心组件

| 组件 | 功能 |
|------|------|
| Controller Manager | 管理所有控制器的生命周期 |
| Hardware Interface | 硬件抽象层，提供read/write接口 |
| Controller | 具体的控制算法（位置、速度、力矩等） |
| ros2_control_node | 控制器管理器的ROS2节点 |

## 控制器类型

- `joint_state_broadcaster`：发布关节状态
- `forward_command_controller`：直接转发命令
- `joint_trajectory_controller`：轨迹跟踪控制器
- `diff_drive_controller`：差速驱动控制器

## 在URDF中配置ros2_control

```xml
<ros2_control name="GazeboSystem" type="system">
  <hardware>
    <plugin>gazebo_ros2_control/GazeboSystem</plugin>
  </hardware>
  <joint name="joint1">
    <command_interface name="position"/>
    <state_interface name="position"/>
    <state_interface name="velocity"/>
  </joint>
</ros2_control>
```
