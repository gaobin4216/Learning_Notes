# ROS2机械臂控制

本章介绍如何在ROS2中使用MoveIt2和ros2_control控制机械臂。MoveIt2是MoveIt的ROS2版本，ros2_control是ros_control的ROS2版本。

## MoveIt2 + ros2_control 整体架构

```
用户界面
  │
  ├── RViz2 (MotionPlanning插件)
  │      │
  │      ↓
  ├── MoveIt2 API (C++/Python)
  │      │
  │      ↓
  ┌──────┴───────────────────────────────────────┐
  │              MoveIt2 (move_group)             │
  │  ┌────────────┐  ┌──────────┐  ┌──────────┐ │
  │  │ 运动规划器  │  │ 碰撞检测 │  │ 运动学   │ │
  │  │  (OMPL)    │  │  (FCL)   │  │  (KDL)   │ │
  │  └─────┬──────┘  └──────────┘  └──────────┘ │
  │        │                                      │
  │        ↓ 关节轨迹 (JointTrajectory)           │
  └────────┼──────────────────────────────────────┘
           │
           ↓ Action: FollowJointTrajectory
  ┌────────┴──────────────────────────────────────┐
  │            ros2_control (控制器层)              │
  │  ┌──────────────────┐  ┌──────────────────┐  │
  │  │ joint_trajectory │  │ joint_state      │  │
  │  │ _controller      │  │ _broadcaster     │  │
  │  └────────┬─────────┘  └──────────────────┘  │
  │           │                                    │
  │           ↓ Hardware Interface                  │
  └───────────┼────────────────────────────────────┘
              │
     ┌────────┴────────┐
     │                 │
     ↓                 ↓
┌─────────┐      ┌─────────┐
│  Gazebo │      │  真实    │
│  仿真   │      │  机械臂  │
└─────────┘      └─────────┘
```

# 一、MoveIt2简介

## MoveIt2是什么？

MoveIt2是MoveIt的ROS2版本，是目前最流行的机器人操作平台，支持：
- 运动规划（Motion Planning）
- 碰撞检测（Collision Detection）
- 运动学求解（Kinematics）
- 抓取规划（Grasping）
- 场景规划（Planning Scene）

## 安装MoveIt2

```bash
sudo apt install ros-jazzy-moveit
```

## 快速体验

MoveIt2自带了示例机器人（Panda机械臂），可以直接体验：

```bash
# 安装示例
sudo apt install ros-jazzy-moveit-resources-panda-description

# 启动MoveIt2 Demo
ros2 launch moveit_setup_assistant setup_assistant.launch.py
```

# 二、MoveIt2 Setup Assistant

## 什么是Setup Assistant？

Setup Assistant是一个图形化工具，用于基于URDF生成MoveIt2所需的配置文件。与ROS1版本类似，但界面和功能有改进。

## 操作步骤

```bash
# 启动Setup Assistant
ros2 launch moveit_setup_assistant setup_assistant.launch.py
```

配置流程：
1. **加载URDF**：选择你的URDF/xacro文件
2. **生成碰撞矩阵**：自动计算哪些连杆之间不会碰撞
3. **添加虚拟关节**：如世界坐标系到机器人基座的连接
4. **添加规划组（Planning Group）**：定义运动组（如"arm"包含哪些关节）
5. **添加机器人姿态**：定义预设姿态（如"home"、"ready"）
6. **添加末端执行器**：定义夹爪
7. **生成配置文件**：输出到新的功能包

## 生成的配置包结构

```
my_robot_moveit_config/
├── config/
│   ├── my_robot.srdf                  # 机器人语义描述
│   ├── kinematics.yaml                # 运动学配置
│   ├── joint_limits.yaml              # 关节限制
│   ├── controllers.yaml               # 控制器配置
│   ├── moveit_controllers.yaml        # MoveIt控制器配置
│   └── pilz_cartesian_limits.yaml     # Pilz规划器配置
├── launch/
│   ├── demo.launch.py                 # Demo启动文件
│   ├── move_group.launch.py           # move_group启动
│   ├── moveit_rviz.launch.py          # RViz2启动
│   └── spawn_controllers.launch.py    # 控制器启动
└── package.xml
```

## 运行Demo

cd ~/ros2_ws && colcon build

# 启动Demo（包含RViz2和MoveIt2）
ros2 launch my_robot_moveit_config demo.launch.py

# 三、MoveIt2编程接口

## 运动规划（Python）

```python
#!/usr/bin/env python3
import rclpy
from rclpy.node import Node
from moveit.planning import MoveItPy
from moveit_msgs.msg import MotionPlanRequest
from geometry_msgs.msg import PoseStamped

class MoveItDemo(Node):
    def __init__(self):
        super().__init__('moveit_demo')
        
        # 初始化MoveIt2
        self.moveit = MoveItPy(node_name="moveit_py")
        self.arm = self.moveit.get_planning_component("arm")
        
        self.get_logger().info('MoveIt2 已初始化')

    def plan_to_pose(self):
        """规划到目标位姿"""
        # 设置目标位姿
        target_pose = PoseStamped()
        target_pose.header.frame_id = "world"
        target_pose.pose.position.x = 0.4
        target_pose.pose.position.y = 0.0
        target_pose.pose.position.z = 0.4
        target_pose.pose.orientation.w = 1.0
        
        # 设置目标
        self.arm.set_goal_state(pose_stamped_msg=target_pose)
        
        # 规划
        result = self.arm.plan()
        
        if result:
            self.get_logger().info('规划成功，执行轨迹...')
            self.moveit.execute("arm", result.trajectory, blocking=True)
        else:
            self.get_logger().warn('规划失败！')

    def plan_to_joint_goal(self):
        """规划到关节目标"""
        joint_goal = {
            "joint1": 0.0,
            "joint2": -0.785,
            "joint3": 0.0,
            "joint4": -2.356,
            "joint5": 0.0,
            "joint6": 1.571,
            "joint7": 0.785,
        }
        
        self.arm.set_goal_state(joint_positions=joint_goal)
        result = self.arm.plan()
        
        if result:
            self.moveit.execute("arm", result.trajectory, blocking=True)

def main(args=None):
    rclpy.init(args=args)
    node = MoveItDemo()
    
    # 规划到目标位姿
    node.plan_to_pose()
    
    rclpy.shutdown()
```

## 使用MoveGroupInterface（C++）

```cpp
#include <rclcpp/rclcpp.hpp>
#include <moveit/move_group_interface/move_group_interface.h>
#include <geometry_msgs/msg/pose.hpp>

int main(int argc, char** argv) {
    rclcpp::init(argc, argv);
    auto node = std::make_shared<rclcpp::Node>("moveit_demo");
    
    // 初始化MoveGroupInterface
    moveit::planning_interface::MoveGroupInterface arm(node, "arm");
    
    // 设置目标位姿
    geometry_msgs::msg::Pose target_pose;
    target_pose.position.x = 0.4;
    target_pose.position.y = 0.0;
    target_pose.position.z = 0.4;
    target_pose.orientation.w = 1.0;
    arm.setPoseTarget(target_pose);
    
    // 规划
    moveit::planning_interface::MoveGroupInterface::Plan plan;
    bool success = (arm.plan(plan) == moveit::core::MoveItErrorCode::SUCCESS);
    
    if (success) {
        RCLCPP_INFO(node->get_logger(), "规划成功，执行轨迹...");
        arm.execute(plan);
    } else {
        RCLCPP_WARN(node->get_logger(), "规划失败！");
    }
    
    rclcpp::shutdown();
    return 0;
}
```

# 四、ros2_control配置

## 在URDF中添加ros2_control标签

```xml
<ros2_control name="my_robot" type="system">
  <hardware>
    <plugin>fake_components/GenericSystem</plugin>
  </hardware>
  
  <joint name="joint1">
    <command_interface name="position">
      <param name="min">-3.14</param>
      <param name="max">3.14</param>
    </command_interface>
    <state_interface name="position"/>
    <state_interface name="velocity"/>
  </joint>
  
  <joint name="joint2">
    <command_interface name="position">
      <param name="min">-3.14</param>
      <param name="max">3.14</param>
    </command_interface>
    <state_interface name="position"/>
    <state_interface name="velocity"/>
  </joint>
</ros2_control>
```

## 控制器配置（YAML）

```yaml
# config/controllers.yaml
controller_manager:
  ros__parameters:
    update_rate: 100  # 100Hz控制频率
    
    joint_state_broadcaster:
      type: joint_state_broadcaster/JointStateBroadcaster
    
    arm_controller:
      type: joint_trajectory_controller/JointTrajectoryController

arm_controller:
  ros__parameters:
    joints:
      - joint1
      - joint2
      - joint3
      - joint4
      - joint5
      - joint6
      - joint7
    
    command_interfaces:
      - position
    
    state_interfaces:
      - position
      - velocity
    
    state_publish_rate: 50.0
    action_monitor_rate: 20.0
    allow_partial_joints_goal: false
```

## 控制器启动文件

```python
# launch/spawn_controllers.launch.py
from launch import LaunchDescription
from launch_ros.actions import Node

def generate_launch_description():
    return LaunchDescription([
        # 启动关节状态广播器
        Node(
            package='controller_manager',
            executable='spawner',
            arguments=['joint_state_broadcaster', '-c', '/controller_manager'],
            output='screen'
        ),
        
        # 启动手臂控制器
        Node(
            package='controller_manager',
            executable='spawner',
            arguments=['arm_controller', '-c', '/controller_manager'],
            output='screen'
        ),
    ])
```

# 五、MoveIt2 + Gazebo仿真实战

## 完整启动流程

```bash
# 1. 启动Gazebo
ros2 launch my_robot_description gazebo.launch.py

# 2. 启动控制器
ros2 launch my_robot_moveit_config spawn_controllers.launch.py

# 3. 启动MoveIt2
ros2 launch my_robot_moveit_config move_group.launch.py

# 4. 启动RViz2
ros2 launch my_robot_moveit_config moveit_rviz.launch.py
```

或者用一个整合的launch文件：
```bash
ros2 launch my_robot_moveit_config demo.launch.py
```

## 与ROS1的关键区别

| 对比项 | ROS1 (MoveIt + ros_control) | ROS2 (MoveIt2 + ros2_control) |
|--------|---------------------------|------------------------------|
| 控制框架 | ros_control | ros2_control |
| 控制器管理 | rosrun controller_manager spawner | ros2 run controller_manager spawner |
| 配置格式 | YAML + XML launch | YAML + Python launch |
| 硬件接口 | RobotHW类 | HardwareInterface类 |
| 仿真插件 | gazebo_ros_control | gazebo_ros2_control |

# 六、真实机械臂驱动

## 驱动真实机械臂的一般步骤

1. **安装驱动包**：`sudo apt install ros-jazzy-xxx-driver`
2. **配置控制器YAML**：设置关节名称、PID参数等
3. **启动控制器**：通过launch文件启动ros2_control
4. **连接MoveIt2**：配置MoveIt2的控制器接口
5. **编程控制**：通过MoveIt2 API发送目标

## 常见机械臂驱动包

| 机械臂 | ROS2驱动包 |
|--------|-----------|
| Universal Robots | `ur_robot_driver` |
| Franka Emika | `franka_ros2` |
| Kinova | `kortex_driver` |
| Dobot | `dobot_ros2` |

## 从仿真到真实硬件

```yaml
# 仿真配置
hardware:
  plugin: fake_components/GenericSystem

# 真实硬件配置（以UR为例）
hardware:
  plugin: ur_robot_driver/URHardware
  parameters:
    robot_ip: "192.168.1.100"
    use_fake_hardware: false
```

只需要修改URDF中的硬件插件配置，其他代码无需改变——这就是ros2_control硬件抽象层的优势。
