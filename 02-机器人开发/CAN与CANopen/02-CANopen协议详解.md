## 1. CANopen概述

### 1.1 什么是CANopen？

```
CANopen = CAN + Open（开放协议）

定义：
- 基于CAN总线的应用层协议
- 由CiA（CAN in Automation）组织维护
- 标准：CiA 301（应用层）、CiA 402（电机驱动）等

特点：
┌─────────────────────────────────────────────────────────────┐
│  ✅ 标准化：统一的设备模型和通信协议                         │
│  ✅ 互操作性：不同厂商设备可以互操作                         │
│  ✅ 灵活性：支持多种通信模式                                 │
│  ✅ 实时性：PDO提供高效数据传输                              │
│  ✅ 可配置性：对象字典支持参数配置                           │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 CAN vs CANopen

```
关系：
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                   应用层                             │   │

│  │  │  - SDO/PDO通信                               │   │   │
│  │  │  - NMT网络管理                               │   │   │
│  │  └─────────────────────────────────────────────┘   │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                   数据链路层                         │   │
│  │  ┌─────────────────────────────────────────────┐   │   │
│  │  │              CAN协议                         │   │   │
│  │  │  - 数据帧/远程帧                             │   │   │
│  │  │  - 总线仲裁                                  │   │   │

│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘

区别：
┌─────────────┬─────────────────┬─────────────────────────────┐
│    特性     │       CAN       │          CANopen            │
├─────────────┼─────────────────┼─────────────────────────────┤

│   数据格式  │   原始数据帧     │     结构化对象字典          │
│   网络管理  │   无             │     NMT状态机               │
│   设备模型  │   无             │     标准化设备描述          │
│   通信模式  │   点对点         │     SDO/PDO                 │
└─────────────┴─────────────────┴─────────────────────────────┘
```

## 2. OSI模型与CANopen

### 2.1 OSI七层模型

```
┌─────────────────────────────────────────────────────────────┐
│  OSI模型        │      CAN/CANopen实现                      │
├─────────────────┼───────────────────────────────────────────┤
│  7. 应用层      │      CANopen（对象字典、SDO/PDO）         │
│  6. 表示层      │      -                                     │
│  5. 会话层      │      -                                     │
│  4. 传输层      │      -                                     │
│  3. 网络层      │      -                                     │
│  2. 数据链路层  │      CAN协议                               │
│  1. 物理层      │      CAN收发器、电缆                       │
└─────────────────┴───────────────────────────────────────────┘

关键点：
- CAN只实现了物理层和数据链路层
- CANopen在CAN之上实现了应用层
- 这种分层设计使得协议灵活可扩展
```

## 3. 对象字典

### 3.1 什么是对象字典？

```
对象字典（Object Dictionary, OD）：
- CANopen设备的核心数据结构
- 存储设备的所有参数和配置
- 通过索引和子索引访问

类比：
┌─────────────────────────────────────────────────────────────┐
│  对象字典 ≈ 设备的"注册表"或"配置文件"                      │
│                                                             │
│  索引（Index）    ≈ 章节号                                   │
│  子索引（SubIndex）≈ 小节号                                  │
│  数据            ≈ 内容                                     │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 对象字典结构

```
对象字典索引分配：

┌─────────────────┬────────────────────────────────────────────┐
│   索引范围      │                    含义                    │
├─────────────────┼────────────────────────────────────────────┤
│   0x0000        │  保留                                      │
│   0x0001-0x001F │  静态数据类型                               │
│   0x0020-0x003F │  复杂数据类型                               │
│   0x0040-0x005F │  制造商特定数据类型                         │
│   0x0060-0x007F │  设备子协议静态数据类型                     │
│   0x0080-0x009F │  设备子协议复杂数据类型                     │
│   0x00A0-0x0FFF │  保留                                      │
│   0x1000-0x1FFF │  通信配置参数                               │
│   0x2000-0x5FFF │  制造商特定参数                             │
│   0x6000-0x9FFF │  标准化设备子协议参数                       │
│   0xA000-0xBFFF │  标准化接口配置参数                         │
│   0xC000-0xFFFF │  保留                                      │
└─────────────────┴────────────────────────────────────────────┘
```

### 3.3 重要索引详解

```
1. 通信参数（0x1000-0x1FFF）

┌─────────────┬────────────────────────────────────────────────┐
│   索引      │                    含义                        │
├─────────────┼────────────────────────────────────────────────┤
│   0x1000    │  设备类型                                      │
│   0x1001    │  错误寄存器                                    │
│   0x1005    │  同步对象COB-ID                                │
│   0x1006    │  通信循环周期                                  │
│   0x1008    │  制造商设备名称                                │
│   0x1009    │  制造商硬件版本                                │
│   0x100A    │  制造商软件版本                                │
│   0x1014    │  紧急对象COB-ID                                │
│   0x1017    │  心跳周期                                      │
│   0x1018    │  标识符对象                                    │
│   0x1400-0x1403 │  RPDO通信参数（0-3）                      │
│   0x1600-0x1603 │  RPDO映射参数（0-3）                      │
│   0x1800-0x1803 │  TPDO通信参数（0-3）                      │
│   0x1A00-0x1A03 │  TPDO映射参数（0-3）                      │
└─────────────┴────────────────────────────────────────────────┘

2. 制造商特定参数（0x2000-0x5FFF）

┌─────────────┬────────────────────────────────────────────────┐
│   索引      │                    含义                        │
├─────────────┼────────────────────────────────────────────────┤
│   0x2000    │  制造商特定参数1                               │
│   0x2001    │  制造商特定参数2                               │
│   ...       │  ...                                           │
│   0x5FFF    │  制造商特定参数n                               │
└─────────────┴────────────────────────────────────────────────┘

3. 标准化设备参数（0x6000-0x9FFF）

┌─────────────┬────────────────────────────────────────────────┐
│   索引      │                    含义                        │
├─────────────┼────────────────────────────────────────────────┤
│   0x6000    │  设备输入数据                                  │
│   0x6200    │  设备输出数据                                  │
│   0x6400    │  模拟量输入                                    │
│   0x6410    │  模拟量输出                                    │
└─────────────┴────────────────────────────────────────────────┘
```

### 3.4 对象字典示例

```
示例：配置电机驱动器

1. 读取设备类型（0x1000）
   SDO读取：601#40 00 10 00 00 00 00 00
   SDO响应：581#43 00 10 00 02 01 00 00
   解析：设备类型 = 0x00000102

2. 配置心跳周期（0x1017）
   SDO写入：601#23 17 10 00 E8 03 00 00
   解析：心跳周期 = 0x03E8 = 1000ms

3. 配置电机控制字（0x6040）
   SDO写入：601#2F 40 60 00 06 00 00 00
   解析：控制字 = 0x0006（准备启动）
```

## 4. CANopen数据帧

### 4.1 数据帧格式

```
CANopen数据帧结构：

┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  CAN帧ID（11位）                                           │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  功能码（4位）  │  节点ID（7位）                    │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  数据域（8字节）                                            │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  字节0  │  字节1  │  字节2  │  ...  │  字节7       │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘

示例：
- COB-ID = 0x601
- 功能码 = 0x6（SDO请求）
- 节点ID = 0x01
- 数据 = 0x40 0x00 0x10 0x00 0x00 0x00 0x00 0x00
```

### 4.2 字节序（小端模式）

```
CANopen使用小端模式（Little-Endian）：

┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  示例：16位数据 0x1234                                      │
│                                                             │
│  内存地址：  0x00    0x01                                    │
│  存储内容：  0x34    0x12                                    │
│              ↑低字节  ↑高字节                                │
│                                                             │
│  CAN帧发送顺序：0x34, 0x12                                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘

示例：32位数据 0x12345678
- 内存：0x78, 0x56, 0x34, 0x12
- CAN帧：0x78, 0x56, 0x34, 0x12

注意：
- 与大多数网络协议（大端）不同
- 解析数据时需要注意字节序
```

### 4.3 DLC（数据长度码）

```
DLC = Data Length Code

作用：
- 指定数据域的实际字节数
- 范围：0~8

示例：
┌─────────────┬────────────────────────────────────────────────┐
│   DLC值     │                    含义                        │
├─────────────┼────────────────────────────────────────────────┤
│     0       │  无数据                                        │
│     1       │  1字节数据                                     │
│     2       │  2字节数据                                     │
│     3       │  3字节数据                                     │
│     4       │  4字节数据                                     │
│     5       │  5字节数据                                     │
│     6       │  6字节数据                                     │
│     7       │  7字节数据                                     │
│     8       │  8字节数据                                     │
└─────────────┴────────────────────────────────────────────────┘

cansend工具：
- 根据输入的十六进制字符数量自动计算DLC
- 例：cansend can0 123#AABB → DLC=2
- 例：cansend can0 123#AABBCCDD → DLC=4
```

## 5. 状态机

### 5.1 CANopen状态机

```
CANopen设备有6种状态：

┌─────────────────────────────────────────────────────────────┐
│                                                             │
│                    ┌─────────────┐                          │
│                    │  初始化     │                          │
│                    │ (Initialisation)                       │
│                    └──────┬──────┘                          │
│                           │                                 │
│                           ▼                                 │
│                    ┌─────────────┐                          │
│                    │  预操作     │                          │
│                    │ (Pre-operational)                      │
│                    └──────┬──────┘                          │
│                           │                                 │
│           ┌───────────────┼───────────────┐                │
│           │               │               │                │
│           ▼               ▼               ▼                │
│    ┌─────────────┐ ┌─────────────┐ ┌─────────────┐        │
│    │  停止       │ │  运行       │ │  复位       │        │
│    │ (Stopped)   │ │ (Operational)│ │ (Reset)    │        │
│    └─────────────┘ └─────────────┘ └─────────────┘        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 状态说明

```
1. 初始化状态（Initialisation）
   - 设备上电后自动进入
   - 初始化硬件和通信
   - 发送Boot-up报文
   - 自动切换到预操作状态

2. 预操作状态（Pre-operational）
   - 可以进行SDO通信
   - 可以配置设备参数
   - 不能进行PDO通信
   - 可以接收NMT命令

3. 运行状态（Operational）
   - 可以进行SDO通信
   - 可以进行PDO通信
   - 设备正常工作
   - 可以接收NMT命令

4. 停止状态（Stopped）
   - 不能进行SDO通信
   - 不能进行PDO通信
   - 只能接收NMT命令
   - 用于设备维护

5. 复位状态（Reset）
   - 重置设备
   - 重新初始化
   - 回到初始化状态
```

### 5.3 状态转换

```
状态转换命令（NMT命令）：

┌─────────────┬─────────────┬────────────────────────────────┐
│   命令      │   代码      │              效果              │
├─────────────┼─────────────┼────────────────────────────────┤
│  启动       │   0x01      │  切换到运行状态                │
│  停止       │   0x02      │  切换到停止状态                │
│  预操作     │   0x80      │  切换到预操作状态              │
│  复位节点   │   0x81      │  复位设备                      │
│  复位通信   │   0x82      │  复位通信参数                  │
└─────────────┴─────────────┴────────────────────────────────┘

NMT命令格式：
- COB-ID = 0x000
- 数据 = [命令代码, 节点ID]
- 节点ID = 0 表示广播（所有节点）

示例：
- 启动节点1：cansend can0 000#0101
- 停止所有节点：cansend can0 000#0200
- 预操作节点2：cansend can0 000#8002
```

## 6. NMT网络管理

### 6.1 NMT概述

```
NMT = Network Management

功能：
- 管理网络中的节点状态
- 控制节点的启动、停止、复位
- 监测节点状态

NMT服务：
┌─────────────┬────────────────────────────────────────────────┐
│   服务      │                    说明                        │
├─────────────┼────────────────────────────────────────────────┤
│  NMT命令    │  主站发送，控制从站状态                        │
│  心跳       │  从站周期性发送，表明存活                      │
│  节点保护   │  主站轮询从站，检查状态                        │
│  Boot-up    │  从站初始化完成时发送                          │
└─────────────┴────────────────────────────────────────────────┘
```

### 6.2 NMT模块控制

```
NMT模块控制报文：

格式：
┌─────────────────────────────────────────────────────────────┐
│  COB-ID: 0x000                                              │
│  数据: [命令代码, 节点ID]                                   │
└─────────────────────────────────────────────────────────────┘

命令代码：
┌─────────────┬────────────────────────────────────────────────┐
│   代码      │                    含义                        │
├─────────────┼────────────────────────────────────────────────┤
│   0x01      │  启动远程节点                                  │
│   0x02      │  停止远程节点                                  │
│   0x80      │  进入预操作状态                                │
│   0x81      │  复位节点                                      │
│   0x82      │  复位通信                                      │
└─────────────┴────────────────────────────────────────────────┘

节点ID：
- 0x00：广播（所有节点）
- 0x01~0x7F：指定节点

示例：
- 启动节点1：cansend can0 000#0101
- 停止所有节点：cansend can0 000#0200
- 复位节点3：cansend can0 000#8103
```

### 6.3 心跳报文

```
心跳报文（Heartbeat）：

作用：
- 从站周期性发送心跳，表明自己存活
- 主站通过心跳检测从站是否在线

格式：
┌─────────────────────────────────────────────────────────────┐
│  COB-ID: 0x700 + 节点ID                                     │
│  数据: [状态字]                                             │
└─────────────────────────────────────────────────────────────┘

状态字：
┌─────────────┬────────────────────────────────────────────────┐
│   值        │                    含义                        │
├─────────────┼────────────────────────────────────────────────┤
│   0x00      │  Boot-up                                       │
│   0x04      │  停止状态                                      │
│   0x05      │  运行状态                                      │
│   0x7F      │  预操作状态                                    │
└─────────────┴────────────────────────────────────────────────┘

配置心跳周期：
- 索引：0x1017
- 单位：ms
- 示例：设置心跳周期为1000ms
  SDO写入：601#23 17 10 00 E8 03 00 00

示例：
- 节点1心跳（预操作）：701#7F
- 节点2心跳（运行）：702#05
```

### 6.4 Boot-up报文

```
Boot-up报文：

触发条件：
- 设备上电初始化完成
- 设备复位后重新初始化完成

格式：
┌─────────────────────────────────────────────────────────────┐
│  COB-ID: 0x700 + 节点ID                                     │
│  数据: 0x00                                                 │
└─────────────────────────────────────────────────────────────┘

作用：
- 通知主站设备已就绪
- 设备自动进入预操作状态
- 主站可以开始配置设备

示例：
- 节点1 Boot-up：701#00
- 节点2 Boot-up：702#00
```

## 7. SDO通信

### 7.1 SDO概述

```
SDO = Service Data Object

作用：
- 读写对象字典中的参数
- 配置设备参数
- 传输大数据（分段传输）

特点：
- 基于请求-响应模式
- 有确认机制
- 适合非实时数据传输
- 最大传输长度无限制（分段传输）

通信模式：
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  主站（Client）          从站（Server）                     │
│      │                       │                              │
│      │    SDO请求            │                              │
│      │──────────────────────►│                              │
│      │                       │                              │
│      │    SDO响应            │                              │
│      │◄──────────────────────│                              │
│      │                       │                              │
└─────────────────────────────────────────────────────────────┘
```

### 7.2 SDO报文格式

```
SDO请求报文（主站→从站）：

┌─────────────────────────────────────────────────────────────┐
│  COB-ID: 0x600 + 节点ID                                     │
│  数据: [命令字, 索引低, 索引高, 子索引, 数据0-4]           │
└─────────────────────────────────────────────────────────────┘

SDO响应报文（从站→主站）：

┌─────────────────────────────────────────────────────────────┐
│  COB-ID: 0x580 + 节点ID                                     │
│  数据: [命令字, 索引低, 索引高, 子索引, 数据0-4]           │
└─────────────────────────────────────────────────────────────┘

命令字类型：
┌─────────────┬─────────────┬────────────────────────────────┐
│   操作      │   命令字    │              说明              │
├─────────────┼─────────────┼────────────────────────────────┤
│  读取       │   0x40      │  请求读取数据                  │
│  写入1字节  │   0x2F      │  写入8位数据                   │
│  写入2字节  │   0x2B      │  写入16位数据                  │
│  写入4字节  │   0x23      │  写入32位数据                  │
│  成功响应   │   0x43/0x4B/0x4F │  读取成功响应            │
│  成功写入   │   0x60      │  写入成功响应                  │
│  错误响应   │   0x80      │  操作失败                      │
└─────────────┴─────────────┴────────────────────────────────┘
```

### 7.3 SDO读取示例

```
示例：读取设备类型（索引0x1000，子索引0x00）

1. 主站发送读取请求
   COB-ID: 0x601
   数据: 0x40 0x00 0x10 0x00 0x00 0x00 0x00 0x00
   解析: 命令字=0x40（读取），索引=0x1000，子索引=0x00

2. 从站响应
   COB-ID: 0x581
   数据: 0x43 0x00 0x10 0x00 0x02 0x01 0x00 0x00
   解析: 命令字=0x43（4字节读取成功），数据=0x00000102

cansend命令：
- 发送：cansend can0 601#4000100000000000
- 接收：candump can0

解读：
- 0x601 = 0x600 + 0x01（节点1的SDO请求）
- 0x581 = 0x580 + 0x01（节点1的SDO响应）
```

### 7.4 SDO写入示例

```
示例：写入心跳周期（索引0x1017，子索引0x00，值1000ms）

1. 主站发送写入请求
   COB-ID: 0x601
   数据: 0x23 0x17 0x10 0x00 0xE8 0x03 0x00 0x00
   解析: 命令字=0x23（4字节写入），索引=0x1017，数据=0x03E8（1000）

2. 从站响应
   COB-ID: 0x581
   数据: 0x60 0x17 0x10 0x00 0x00 0x00 0x00 0x00
   解析: 命令字=0x60（写入成功）

cansend命令：
- 发送：cansend can0 601#23171000E8030000
- 接收：candump can0

解读：
- 0x23 = 写入32位数据
- 0x17 0x10 = 索引0x1017（小端）
- 0xE8 0x03 = 1000（小端）
```

## 8. PDO通信

### 8.1 PDO概述

```
PDO = Process Data Object

作用：
- 实时传输过程数据
- 高效、低延迟
- 适合周期性数据传输

特点：
┌─────────────────────────────────────────────────────────────┐
│  ✅ 无确认机制：发送后不等待响应                            │
│  ✅ 高效率：数据直接映射到对象字典                          │
│  ✅ 低延迟：适合实时控制                                    │
│  ✅ 最大8字节：受限于CAN帧数据长度                          │
└─────────────────────────────────────────────────────────────┘

TPDO vs RPDO：
┌─────────────┬────────────────────────────────────────────────┐
│   类型      │                    说明                        │
├─────────────┼────────────────────────────────────────────────┤
│  TPDO       │  发送PDO，从站发送数据到主站                   │
│  RPDO       │  接收PDO，从站接收主站发送的数据               │
└─────────────┴────────────────────────────────────────────────┘

每个节点最多4路TPDO和4路RPDO：
- TPDO0~TPDO3
- RPDO0~RPDO3
```

### 8.2 PDO通信参数

```
PDO通信参数定义PDO的通信属性：

TPDO通信参数（索引0x1800~0x1803）：
┌─────────────┬────────────────────────────────────────────────┐
│   子索引    │                    含义                        │
├─────────────┼────────────────────────────────────────────────┤
│     0       │  映射对象数                                    │
│     1       │  COB-ID                                        │
│     2       │  传输类型                                      │
│     3       │  禁止时间                                      │
│     5       │  事件定时器                                    │
└─────────────┴────────────────────────────────────────────────┘

RPDO通信参数（索引0x1400~0x1403）：
┌─────────────┬────────────────────────────────────────────────┐
│   子索引    │                    含义                        │
├─────────────┼────────────────────────────────────────────────┤
│     0       │  映射对象数                                    │
│     1       │  COB-ID                                        │
│     2       │  传输类型                                      │
└─────────────┴────────────────────────────────────────────────┘

传输类型：
┌─────────────┬────────────────────────────────────────────────┐
│   值        │                    含义                        │
├─────────────┼────────────────────────────────────────────────┤
│   0         │  非同步，事件触发                              │
│   1~240     │  同步，每N个SYNC发送                           │
│   253       │  远程请求触发                                  │
│   254       │  事件触发（制造商特定）                        │
│   255       │  事件触发（设备子协议）                        │
└─────────────┴────────────────────────────────────────────────┘
```

### 8.3 PDO映射参数

```
PDO映射参数定义PDO数据的内容：

TPDO映射参数（索引0x1A00~0x1A03）：
RPDO映射参数（索引0x1600~0x1603）：

格式：
┌─────────────┬────────────────────────────────────────────────┐
│   子索引    │                    含义                        │
├─────────────┼────────────────────────────────────────────────┤
│     0       │  映射对象数N                                   │
│    1~N      │  映射条目                                      │
└─────────────┴────────────────────────────────────────────────┘

映射条目格式（32位）：
┌─────────────────────────────────────────────────────────────┐
│  [31:24] 数据长度（位）                                      │
│  [23:16] 子索引                                              │
│  [15:0]  对象字典索引                                        │
└─────────────────────────────────────────────────────────────┘

示例：映射两个变量到TPDO0
- 0x1A00.sub0 = 2（2个映射项）
- 0x1A00.sub1 = 0x60000108（索引0x6000，子索引1，8位）
- 0x1A00.sub2 = 0x60010010（索引0x6001，子索引0，16位）
```

### 8.4 PDO配置示例

```
配置TPDO0（节点ID=0x05）：

1. 禁用TPDO0
   SDO写入：605#23 00 18 00 80 05 00 80
   解析：COB-ID = 0x80000185（禁用位=1）

2. 设置传输类型（事件触发）
   SDO写入：605#2F 01 18 00 FE 00 00 00
   解析：传输类型 = 254（事件触发）

3. 设置禁止时间（10ms）
   SDO写入：605#2B 03 18 00 64 00 00 00
   解析：禁止时间 = 100（×100µs = 10ms）

4. 清除原有映射
   SDO写入：605#2F 00 1A 00 00 00 00 00
   解析：映射对象数 = 0

5. 添加映射项1
   SDO写入：605#23 01 1A 00 00 60 00 08
   解析：索引0x6000，子索引0，8位

6. 添加映射项2
   SDO写入：605#23 02 1A 00 01 60 00 10
   解析：索引0x6001，子索引0，16位

7. 设置映射对象数
   SDO写入：605#2F 00 1A 00 02 00 00 00
   解析：映射对象数 = 2

8. 启用TPDO0
   SDO写入：605#23 00 18 00 85 01 00 00
   解析：COB-ID = 0x00000185（启用）
```

### 8.5 PDO报文示例

```
TPDO0报文（节点ID=0x05）：

COB-ID: 0x185 = 0x180 + 0x05
数据: 0xAA 0xBB 0xCC

解析：
- 0xAA = 索引0x6000的数据（8位）
- 0xBB 0xCC = 索引0x6001的数据（16位，小端）

cansend命令：

- 接收：candump can0

```

## 9. 常见问题

### Q1: SDO和PDO有什么区别？

**A**:
| 特性 | SDO | PDO |
|------|-----|-----|
| 确认机制 | 有（请求-响应） | 无（发送即忘） |
| 数据长度 | 无限制（分段） | 最大8字节 |
| 实时性 | 差 | 好 |
| 用途 | 配置参数 | 实时数据 |

### Q2: 如何选择TPDO和RPDO？

**A**:
- **TPDO**：从站发送数据到主站（如传感器数据、状态）
- **RPDO**：从站接收主站数据（如控制指令、目标值）

### Q3: 对象字典索引怎么找？

**A**:
1. 查阅设备手册
2. 使用CANopen配置工具
3. 通过SDO扫描

### Q4: 波特率怎么设置？

**A**:
1. 所有节点必须使用相同波特率
2. 根据传输距离选择
3. 常用：500kbps（汽车）、1Mbps（高速）

### Q5: 如何调试CANopen通信？

**A**:
1. 使用candump监控总线
2. 使用cansend发送测试帧
3. 使用CANopen配置工具
4. 检查错误计数器
