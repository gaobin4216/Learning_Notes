## 1. CAN概述

### 1.1 什么是CAN？

```
CAN = Controller Area Network（控制器局域网络）

历史：
- 1986年：德国BOSCH公司开发
- 1993年：成为国际标准ISO 11898
- 现在：汽车、工业控制、医疗设备等领域广泛使用

特点：
┌─────────────────────────────────────────────────────────────┐
│  ✅ 多主站结构：任何节点都可以主动发送数据                    │
│  ✅ 差分信号：抗干扰能力强                                   │
│  ✅ 高可靠性：内置错误检测和处理机制                          │
│  ✅ 实时性好：最高1Mbps传输速率                              │
│  ✅ 成本低：只需两根信号线                                   │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 CAN vs 其他通信协议

| 特性 | CAN | RS485 | Ethernet |
|------|-----|-------|----------|
| 拓扑结构 | 多主站 | 主从 | 星形 |
| 最大速率 | 1Mbps | 10Mbps | 1Gbps+ |
| 抗干扰 | 强（差分） | 强（差分） | 中 |
| 线缆数量 | 2根 | 2根 | 4根/8根 |
| 实时性 | 好 | 中 | 差 |
| 成本 | 低 | 低 | 高 |
| 典型应用 | 汽车、工业 | 工业仪表 | 互联网 |

### 1.3 应用场景

```
1. 汽车电子
   ├── 发动机控制
   ├── 变速箱控制
   ├── ABS防抱死
   ├── 车身电子
   └── 诊断系统（OBD）

2. 工业自动化
   ├── PLC通信
   ├── 伺服电机控制
   ├── 传感器网络
   └── 机器人控制

3. 医疗设备
   ├── 监护仪
   ├── 输液泵
   └── 手术机器人

4. 航空航天
   ├── 飞行控制
   └── 发动机监控
```

## 2. 硬件连接

### 2.1 CAN节点组成

```
一个CAN节点由以下部分组成：

┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  ┌─────────┐      CAN_Tx      ┌─────────┐     CAN_High     │
│  │         │──────────────────►│         │──────────────────►│
│  │  CAN    │      CAN_Rx      │  CAN    │                   │──── CAN总线
│  │控制器   │◄──────────────────│ 收发器  │                   │
│  │         │                   │         │──────────────────►│
│  └─────────┘                   └─────────┘     CAN_Low      │
│                                                             │
└─────────────────────────────────────────────────────────────┘

- CAN控制器：处理协议、收发数据
- CAN收发器：电平转换（TTL ↔ 差分信号）
- CAN_High / CAN_Low：差分信号线
```

### 2.2 总线拓扑

```
方式1：开环总线（ISO 11898-2）

    ┌───────────────────────────────────────────────┐
    │                                               │
    │  ┌─────┐    ┌─────┐    ┌─────┐    ┌─────┐   │
    │  │节点1│    │节点2│    │节点3│    │节点4│   │
    │  └──┬──┘    └──┬──┘    └──┬──┘    └──┬──┘   │
    │     │          │          │          │       │
    ├─────┴──────────┴──────────┴──────────┴───────┤
    │                   CAN_High                    │
    ├───────────────────────────────────────────────┤
    │                   CAN_Low                     │
    ├─────┬──────────┬──────────┬──────────┬───────┤
    │     │          │          │          │       │
    │  ┌──┴──┐    ┌──┴──┐    ┌──┴──┐    ┌──┴──┐   │
    │  │120Ω │    │     │    │     │    │120Ω │   │
    │  └─────┘    └─────┘    └─────┘    └─────┘   │
    │  终端电阻                              终端电阻│
    └───────────────────────────────────────────────┘

特点：
- 两端必须有120Ω终端电阻
- 最大传输距离：1Mbps时40m，10kbps时6km
- 最多节点数：理论无限制，实际建议≤30
```

### 2.3 终端电阻

```
为什么需要120Ω终端电阻？

1. 阻抗匹配
   - CAN电缆特性阻抗约120Ω
   - 终端电阻匹配阻抗，避免信号反射

2. 信号完整性
   - 无终端电阻：信号振铃、过冲
   - 有终端电阻：信号干净、稳定

3. 功耗考虑
   - 120Ω是信号完整性和功耗的平衡点

计算公式：
R_terminal = Z_cable（电缆特性阻抗）

常见问题：
┌─────────────────────────────────────────────────────────────┐
│  Q: 终端电阻放在哪里？                                       │
│  A: 总线两端，每个端点一个120Ω电阻                          │
│                                                             │
│  Q: 可以用两个60Ω串联吗？                                    │
│  A: 可以，效果相同                                           │
│                                                             │
│  Q: 中间节点需要终端电阻吗？                                 │
│  A: 不需要，只需两端                                         │
└─────────────────────────────────────────────────────────────┘
```

### 2.4 接线规范

```
推荐线缆：
- 双绞线（CAN_High和CAN_Low绞合）
- 特性阻抗120Ω
- 截面积0.35~0.75mm²

接线注意事项：
┌─────────────────────────────────────────────────────────────┐
│  ✅ 使用双绞线，减少电磁干扰                                 │
│  ✅ 线缆长度匹配，避免时序偏差                               │
│  ✅ 远离强电线路，减少耦合干扰                               │
│  ✅ 屏蔽层单点接地，避免地环路                               │
│  ❌ 避免星形拓扑，使用总线拓扑                               │
│  ❌ 避免线缆过长，超过最大传输距离                           │
└─────────────────────────────────────────────────────────────┘
```

## 3. 差分信号原理

### 3.1 什么是差分信号？

```
差分信号：使用两根线传输一个信号

┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  CAN_High ─────────────────────────────────────────────────  │
│              ↕ 差值 = 有效信号                               │
│  CAN_Low  ─────────────────────────────────────────────────  │
│                                                             │
│  逻辑"0"（显性）：CAN_High - CAN_Low = 2V                   │
│  逻辑"1"（隐性）：CAN_High - CAN_Low = 0V                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 为什么抗干扰强？

```
干扰耦合原理：

┌─────────────────────────────────────────────────────────────┐
│                                                             │
│     噪声源                                                  │
│        ↓                                                    │
│  ┌─────────────────────────────────────────────┐           │
│  │  CAN_High: V_signal + V_noise               │           │
│  │  CAN_Low:  V_signal + V_noise               │           │
│  └─────────────────────────────────────────────┘           │
│        ↓                                                    │
│  接收端计算差值：                                            │
│  (V_signal + V_noise) - (V_signal + V_noise) = 0           │
│                                                             │
│  结论：共模噪声被完全抵消！                                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘

优势：
1. 抗共模干扰：噪声同时耦合到两根线，差值不变
2. 抗电磁干扰：双绞线结构抵消磁场影响
3. 抗地环路：不依赖地线参考
```

### 3.3 电平标准

```
CAN 2.0B电平标准：

┌─────────────┬─────────────┬─────────────┬──────────────────┐
│   状态      │  CAN_High   │   CAN_Low   │     差值         │
├─────────────┼─────────────┼─────────────┼──────────────────┤
│  显性(0)    │   3.5V      │   1.5V      │     2V           │
│  隐性(1)    │   2.5V      │   2.5V      │     0V           │
└─────────────┴─────────────┴─────────────┴──────────────────┘

显性电平（逻辑0）：
- 差值约2V
- 优先级高（总线仲裁）

隐性电平（逻辑1）：
- 差值约0V
- 优先级低
```

### 3.4 半双工通信

```
CAN是半双工通信：

┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  时间轴：                                                    │
│  ─────────────────────────────────────────────────────────  │
│     ↑发送      ↑接收      ↑发送      ↑接收                  │
│     节点A      节点A      节点B      节点B                  │
│                                                             │
│  特点：                                                      │
│  - 同一时刻只能有一个节点发送                                │
│  - 所有节点都可以接收                                        │
│  - 发送和接收不能同时进行                                    │
│                                                             │
│  总线仲裁：                                                  │
│  - 多个节点同时发送时，优先级高的节点获胜                    │
│  - 优先级由报文ID决定（ID越小优先级越高）                    │
│  - 仲裁失败的节点自动退出，等待下次发送                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 4. CAN数据帧

### 4.1 帧类型

```
CAN协议定义了4种帧类型：

┌─────────────┬────────────────────────────────────────────────┐
│   帧类型    │                    用途                        │
├─────────────┼────────────────────────────────────────────────┤
│  数据帧     │  发送数据，最常用的帧类型                      │
│  远程帧     │  请求其他节点发送数据                          │
│  错误帧     │  检测到错误时发送（硬件自动处理）              │
│  过载帧     │  请求延迟发送（硬件自动处理）                  │
└─────────────┴────────────────────────────────────────────────┘

对于一般使用者，只需掌握数据帧和远程帧
```

### 4.2 数据帧结构

```
标准数据帧格式（11位ID）：

┌─────┬────────┬─────┬─────┬─────────────────┬─────┬─────┐
│ SOF │ 仲裁段 │控制段│保留 │     数据段      │CRC段│ EOF │
│ 1位 │  11位  │ 6位 │ 2位 │   0~64位(8字节) │15位 │ 2位 │
└─────┴────────┴─────┴─────┴─────────────────┴─────┴─────┘

扩展数据帧格式（29位ID）：

┌─────┬─────────────────────────────┬─────┬─────────────────┬─────┬─────┐
│ SOF │         仲裁段              │控制段│     数据段      │CRC段│ EOF │
│ 1位 │    11位 + 18位 = 29位      │ 6位 │   0~64位(8字节) │15位 │ 2位 │
└─────┴─────────────────────────────┴─────┴─────────────────┴─────┴─────┘
```

### 4.3 各字段详解

```
1. SOF（Start of Frame）
   - 1位显性电平
   - 表示帧开始

2. 仲裁段
   - 标准帧：11位ID + RTR位
   - 扩展帧：11位ID + SRR + IDE + 18位ID + RTR位
   - ID决定优先级（越小越优先）

3. 控制段
   - IDE：标识符扩展位
   - r0：保留位
   - DLC：数据长度码（0~8）

4. 数据段
   - 0~8字节数据
   - 高位先发（MSB）

5. CRC段
   - 15位CRC校验
   - 1位CRC界定符

6. ACK段
   - 1位ACK槽
   - 1位ACK界定符

7. EOF（End of Frame）
   - 7位隐性电平
   - 表示帧结束
```

### 4.4 比特率与波特率

```
比特率 vs 波特率：

┌─────────────────────────────────────────────────────────────┐
│  比特率：单位时间内传输的二进制位数（bps）                   │
│  波特率：单位时间内传输的符号数（Baud）                      │
│                                                             │
│  关系：比特率 = 波特率 × 每符号比特数                        │
│                                                             │
│  CAN总线：                                                   │
│  - 物理层只有两种状态（显性/隐性）                           │
│  - 每个符号代表1个比特                                       │
│  - 所以：比特率 = 波特率                                     │
└─────────────────────────────────────────────────────────────┘

常用波特率：
┌─────────────┬────────────────────────────────────────────────┐
│   波特率    │                    应用场景                    │
├─────────────┼────────────────────────────────────────────────┤
│   1Mbps     │  高速通信，短距离                              │
│   500kbps   │  汽车CAN总线                                   │
│   250kbps   │  工业控制                                      │
│   125kbps   │  低速应用                                      │
│   100kbps   │  CANopen标准                                   │
└─────────────┴────────────────────────────────────────────────┘
```

## 5. 通信过程

### 5.1 发送过程

```
节点发送数据的完整过程：

┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  1. 应用层准备数据                                           │
│     - 准备要发送的数据（0~8字节）                            │
│     - 指定目标ID                                            │
│                                                             │
│  2. CAN控制器处理                                            │
│     - 封装成CAN帧                                           │
│     - 添加SOF、CRC、EOF等字段                               │
│     - 检查总线状态                                          │
│                                                             │
│  3. 总线仲裁（如果多个节点同时发送）                         │
│     - 监听总线                                              │
│     - 发送ID，比较优先级                                    │
│     - 优先级高的继续发送                                    │
│     - 优先级低的退出，等待下次                              │
│                                                             │
│  4. CAN收发器发送                                            │
│     - 将TTL信号转换为差分信号                                │
│     - 通过CAN_High和CAN_Low发送到总线                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 接收过程

```
节点接收数据的完整过程：

┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  1. CAN收发器接收                                            │
│     - 从总线接收差分信号                                     │
│     - 转换为TTL信号                                         │
│     - 发送到CAN控制器                                       │
│                                                             │
│  2. CAN控制器处理                                            │
│     - 解析CAN帧                                             │
│     - CRC校验                                               │
│     - 提取ID和数据                                          │
│                                                             │
│  3. 过滤器匹配                                               │
│     - 检查ID是否匹配过滤器                                  │
│     - 匹配则接收，不匹配则丢弃                              │
│                                                             │
│  4. 应用层读取                                               │
│     - 从接收缓冲区读取数据                                  │
│     - 处理接收到的信息                                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 5.3 过滤器配置

```
过滤器的作用：
- 只接收需要的报文，减少CPU负担
- 提高系统效率

过滤器类型：
┌─────────────────────────────────────────────────────────────┐
│  1. ID过滤器                                                │
│     - 指定接收的ID范围                                      │
│     - 例：只接收ID=0x100的报文                              │
│                                                             │
│  2. 掩码过滤器                                               │
│     - 指定ID的哪些位需要匹配                                │
│     - 例：接收ID高4位=0x1的所有报文                         │
│                                                             │
│  3. 范围过滤器                                               │
│     - 指定ID的范围                                          │
│     - 例：接收0x100~0x1FF的所有报文                         │
└─────────────────────────────────────────────────────────────┘

示例：
// 只接收ID=0x123的报文
filter.id = 0x123;
filter.mask = 0x7FF;  // 所有位都匹配

// 接收ID高8位=0x12的所有报文
filter.id = 0x1200;
filter.mask = 0x7F00;  // 只匹配高8位
```

## 6. 常见问题

### Q1: CAN总线最大传输距离？

**A**: 取决于波特率：
- 1Mbps：最大40m
- 500kbps：最大100m
- 250kbps：最大250m
- 125kbps：最大500m
- 100kbps：最大6km

### Q2: 为什么需要120Ω终端电阻？

**A**: 
1. 匹配电缆特性阻抗（120Ω）
2. 防止信号反射
3. 保证信号完整性

### Q3: CAN总线最多能接多少节点？

**A**: 
- 理论上无限制
- 实际建议≤30个节点
- 取决于收发器驱动能力和总线负载

### Q4: 显性电平和隐性电平有什么区别？

**A**:
- 显性电平（逻辑0）：差值约2V，优先级高
- 隐性电平（逻辑1）：差值约0V，优先级低
- 总线仲裁时，显性电平会覆盖隐性电平

### Q5: 如何选择CAN波特率？

**A**:
1. 根据传输距离选择
2. 根据实时性要求选择
3. 所有节点必须使用相同波特率
4. 常用：500kbps（汽车）、1Mbps（高速）
